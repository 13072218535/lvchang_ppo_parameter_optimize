# 铝电解槽参数优化 — 多强化学习算法对比方案

## 一、背景与目标

当前项目使用 **MPD-PPO**（Model Predictive Decoding PPO，轨迹感知PPO）作为参数优化算法，其特点为：

- 28维完整轨迹输出（14天×2动作），消除训练/推理分布不一致
- 差异化裁剪（ALF ε=0.1, OUT ε=0.2）
- 滚动时域重规划（每步重新规划14天，仅执行day-1）
- 多步加权奖励（时间衰减+平滑惩罚+边界惩罚）

为验证MPD-PPO相比其他主流强化学习算法的优势，需在**相同环境、相同状态/动作空间、相同奖励函数**的条件下，对以下算法进行横向对比：

| 算法 | 类型 | 策略 | 经验利用 |
|------|------|------|---------|
| **MPD-PPO** | On-policy | 随机策略（对角高斯） | 无回放，逐批更新 |
| **Vanilla PPO** | On-policy | 随机策略（对角高斯） | 无回放，单一裁剪率 |
| **A2C** | On-policy | 随机策略（对角高斯） | 无回放，无裁剪 |
| **DDPG** | Off-policy | 确定性策略+高斯噪声 | 经验回放池 |
| **TD3** | Off-policy | 确定性策略+目标平滑噪声 | 经验回放池 |
| **SAC** | Off-policy | 随机策略+自动熵调节 | 经验回放池 |

## 二、统一实验框架设计

### 2.1 共享组件（无需改动）

所有算法共用以下组件，确保对比的公平性：

```
共享层：
├── VoltageControlEnv          # 仿真环境（28维动作→14天预测→滚动时域）
├── ConditionalVoltagePredictor # 条件电压预测模型（已训练，冻结）
├── 数据加载 & Scaler           # load_data_for_ppo() + StandardScaler
├── 奖励函数                    # R_acc(κ=15) + P_smooth + P_bound
├── 状态空间                    # (7×12 + 14 + 1) = 99维
├── 动作空间                    # 28维，归一化到[-1,1]
└── 超参数                      # 学习率、折扣因子等统一基线
```

### 2.2 Actor网络架构（所有算法共用）

当前MPD-PPO的Actor网络结构可直接复用：

```
State(99) → 拆分 →
  过去7天特征(84)  → LSTM(12+16→128, 2层) → hidden(128) ─┐
  目标电压(14) ──────────────────────────────────────────┤→ concat(158)
  槽嵌入(16) ───────────────────────────────────────────┘      │
                                                               ↓
                                                        SharedNet:
                                                          FC(158→128)→Tanh
                                                          FC(128→64)→Tanh
                                                               │
                                          ┌────────────────────┴──────────────────┐
                                     mean_head(64→28)                    std_head(64→28)
                                        tanh→[-1,1]                     softplus→(0,∞)
```

**算法差异**：
- **PPO/A2C/SAC**：使用完整网络（mean + std），随机策略采样
- **DDPG/TD3**：仅使用 mean_head（无std），确定性策略 + 外部噪声

### 2.3 Critic网络架构（按算法类型区分）

| 算法 | Critic类型 | 网络结构 | 输出 |
|------|-----------|---------|------|
| **MPD-PPO** | V-Critic | LSTM + SharedNet(158→128→64→1) | V(s) ∈ [-50,50] |
| **Vanilla PPO** | V-Critic | 同上 | V(s) ∈ [-50,50] |
| **A2C** | V-Critic | 同上 | V(s) ∈ [-50,50] |
| **DDPG** | Q-Critic | LSTM → concat(s,a) → FC(158+28→128→64→1) | Q(s,a) |
| **TD3** | Twin Q-Critic | 同上 ×2 | Q₁(s,a), Q₂(s,a) |
| **SAC** | Twin Q-Critic + α | 同上 ×2 + log_α参数 | Q₁(s,a), Q₂(s,a), α |

## 三、各算法实现要点

### 3.1 MPD-PPO（当前算法，基准）

**文件**：`ppo.py`（现有）

**核心特征**：
- 差异化裁剪：ALF维度 ε=0.1，OUT维度 ε=0.2
- 独立裁剪+求和：`loss = -(alf_surr + out_surr)`
- GAE优势估计，reward标准化
- Critic输出clamp [-50,50]

**保留**：作为对比基准，不做修改。

### 3.2 Vanilla PPO

**文件**：新建 `algorithms/ppo_vanilla.py`

**与MPD-PPO的差异**：
- 单一裁剪率 ε=0.2（所有28维统一）
- 标准PPO loss：`loss = -min(ratio×A, clip(ratio, 1-ε, 1+ε)×A).mean()`
- 无差异化裁剪，无独立求和

**实现**：继承或复用MPDPPO类的`compute_gae()`和buffer管理，仅修改`update()`中的loss计算。

```python
# 核心差异：统一裁剪替代独立裁剪
ratio = torch.exp(new_log_probs - old_log_probs)
clipped_ratio = torch.clamp(ratio, 1 - 0.2, 1 + 0.2)
surr = torch.min(ratio * adv, clipped_ratio * adv).mean()
actor_loss = -surr
```

### 3.3 A2C（Advantage Actor-Critic）

**文件**：新建 `algorithms/a2c.py`

**核心特征**：
- 无裁剪，纯策略梯度：`loss = -log_prob × A`
- 可加熵正则项：`loss += β × entropy`
- 与PPO共享网络结构，仅update逻辑不同

**实现**：
```python
# A2C核心：无裁剪的策略梯度
actor_loss = -(new_log_probs * advantages).mean()
# 可选熵正则
entropy = dist.entropy().mean()
actor_loss -= 0.01 * entropy  # β=0.01
```

**注意**：A2C对步长敏感，可能需要调低学习率或增加PPO_STEPS_PER_UPDATE来减少方差。

### 3.4 DDPG（Deep Deterministic Policy Gradient）

**文件**：新建 `algorithms/ddpg.py`

**核心组件**：
1. **Actor（确定性）**：`state → mean(28)`，无std头，输出直接作为动作
2. **Critic（Q函数）**：`(state, action) → Q-value`
3. **Target Networks**：Actor和Critic各有一个target网络，软更新 τ=0.005
4. **Replay Buffer**：容量 100,000 transitions
5. **探索噪声**：Ornstein-Uhlenbeck或高斯噪声 σ=0.2

**交互与更新流程**：
```
交互（每步）：
  action = actor(state) + noise      # 确定性策略+噪声
  (s, a, r, s', done) → replay buffer

更新（每步）：
  从buffer采样batch=256
  target_Q = r + γ × Q_target(s', actor_target(s'))
  critic_loss = MSE(Q(s,a), target_Q)
  actor_loss = -Q(s, actor(s)).mean()
  软更新target: θ_target = τ×θ + (1-τ)×θ_target
```

**Critic网络结构**：
```
State(99) → LSTM编码 → hidden(128) ─┐
Action(28) ─────────────────────────┤→ concat(156) → FC(128→64→1) → Q(s,a)
```

### 3.5 TD3（Twin Delayed DDPG）

**文件**：新建 `algorithms/td3.py`

**相比DDPG的3项改进**：
1. **Clipped Double Q-learning**：双Q网络取min，抑制过估计
2. **Delayed Policy Updates**：Critic更新2次才更新1次Actor
3. **Target Policy Smoothing**：target动作加clip噪声，平滑Q值估计

**更新流程**：
```
target_action = actor_target(s') + clip(N(0, σ), -c, c)  # 平滑噪声
target_Q = r + γ × min(Q1_target(s', target_action), Q2_target(s', target_action))
critic_loss = MSE(Q1(s,a), target_Q) + MSE(Q2(s,a), target_Q)

if step % policy_delay == 0:   # policy_delay=2
    actor_loss = -Q1(s, actor(s)).mean()
    更新actor
    软更新target networks
```

### 3.6 SAC（Soft Actor-Critic）

**文件**：新建 `algorithms/sac.py`

**核心特征**：
- 最大熵框架：最大化 `Σ [r + α×H(π)]`
- 自动温度调节：动态调整α以维持目标熵
- 双Q网络（同TD3）
- 随机策略（同PPO）：mean + std → 采样

**温度自动调节**：
```
α_loss = -α × (log_prob + target_entropy).mean()
target_entropy = -action_dim  # -28 for our case
```

**Critic更新**：
```
target_Q = r + γ × (min(Q1_target, Q2_target) - α × log_prob_next)
critic_loss = MSE(Q1, target_Q) + MSE(Q2, target_Q)
```

**Actor更新**（重参数化采样）：
```
action, log_prob = actor.rsample(state)  # 重参数化
actor_loss = (α × log_prob - Q1(state, action)).mean()
```

### 4.1 抽象基类设计

```python
# algorithms/base_algorithm.py
class BaseAlgorithm(ABC):
    """所有对比算法的统一接口"""
    
    @abstractmethod
    def select_action(self, state: np.ndarray) -> np.ndarray:
        """选择动作（28维，[-1,1]）"""
        pass
    
    @abstractmethod
    def store_transition(self, state, action, reward, next_state, done):
        """存储经验"""
        pass
    
    @abstractmethod
    def update(self) -> tuple:
        """更新网络，返回 (actor_loss, critic_loss)"""
        pass
    
    @abstractmethod
    def save_model(self, path: str):
        pass
    
    @abstractmethod
    def load_model(self, path: str):
        pass
```

## 八、预期结果与假设

### 8.1 假设

1. **MPD-PPO应最稳定**：差异化裁剪对ALF/OUT的不同敏感度提供更好的约束
2. **Vanilla PPO可能收敛更快但不够稳定**：单一裁剪率导致某维度过拟合
3. **A2C方差应最大**：无裁剪导致策略更新不稳定
4. **DDPG/TD3可能采样效率更高**：经验回放提高数据利用率
5. **SAC可能接近MPD-PPO**：熵正则化提供类似的探索-利用平衡
6. **Off-policy算法可能过度利用**：在仿真环境中容易找到reward hacking的漏洞

### 8.2 验证方式

训练200-500轮后：
- Reward曲线是否呈现健康的上升趋势
- P_smooth是否控制在不爆炸的范围（>-200）
- P_bound是否有非零值
- 电压MAE是否在合理范围（<200mV）
- ALF/OUT调整是否平滑（日变化在允许范围内）

## 九、注意事项

1. **随机种子固定**：所有算法使用相同随机种子（SEED=42）
2. **模型初始化一致**：Actor LSTM编码器使用相同的初始化策略
3. **离线评估**：visualize时使用 `actor.eval()`，关闭噪声/探索
4. **多次运行取平均**：如有计算资源，每个算法运行3次取平均（不同种子42/123/456）
5. **环境完全一致**：相同的数据划分、scaler、条件预测器
6. **Off-policy交互**：DDPG/TD3/SAC每步都做一次update（与PPO的批更新模式不同），需要在train_compare.py中适配交互逻辑
