## 直接回答：原模型权重不能直接加载，但可以部分复用（迁移学习）

您的问题非常关键。将现有预测模型改造为“条件电压预测器”（输入增加未来动作序列）后，**原始训练好的模型权重不能直接加载使用**，因为输入维度发生了根本变化。但并非完全“无用”——通过**迁移学习**，可以保留并复用模型中的大部分参数（LSTM层、全连接层等），仅重新训练输入层或扩展部分，从而大大加速新模型的收敛。

---

### 一、为什么不能直接加载？

| 原模型                                               | 新模型（条件电压预测器）                                    |
| ---------------------------------------------------- | ----------------------------------------------------------- |
| 输入维度：84（过去7天特征）                          | 输入维度：84 + 28 = 112（84个历史特征 + 未来14天×2个动作） |
| 第一层线性层（或嵌入层）的权重形状：`(84, hidden)` | 第一层权重形状必须为 `(112, hidden)`                      |

**直接加载会报错**：`RuntimeError: size mismatch for fc1.weight: shape (hidden,84) vs (hidden,112)`

即使您使用的是LSTM，其输入维度 `input_size` 也不匹配，无法加载权重。

---

### 二、如何部分复用？——迁移学习策略

您的模型结构为：

```
输入(7, in_dim) → LSTM(hidden=128, layers=2) → BN → FC(128→64→32→1)
```

其中 `in_dim` 当前为84，需要改为112。

**可行方案**：

1. **仅扩充输入投影层**（如果LSTM的 `input_size` 与 `in_dim` 绑定）：

   - 创建新模型，设置 `in_dim=112`。
   - 从原模型加载LSTM层、BN层、全连接层的权重（这些层的形状未变）。
   - 对于LSTM的输入投影权重（形状为 `(4*hidden, in_dim)`），由于 `in_dim` 改变，旧权重无法匹配，**只能随机初始化输入投影部分**。
   - 然后在小学习率下微调整个模型。
2. **更优雅的方式：不改变原有LSTM结构，将未来动作作为条件特征注入**
   保持原有模型输入维度84不变，将未来动作序列通过另一个小型网络编码后，与LSTM的输出拼接。这样**原模型权重完全可复用**，只需额外训练一个小的“条件编码器”。这是推荐做法。

---

### 三、推荐做法：保持原模型，增加条件编码器

#### 架构设计

```
过去7天特征 (batch,7,84) → 原LSTM模型 → 输出隐藏状态 (batch,128)
                                           ↓
未来动作序列 (batch,14,2) → 小型MLP/Conv1D → 条件向量 (batch,32)
                                           ↓
                                    拼接 (128+32=160)
                                           ↓
                                    全连接层 → 预测未来14天电压
```

- **原LSTM模型权重完全保留**（冻结或微调），无需重新训练。
- 新增的条件编码器（例如两层卷积或LSTM处理未来动作序列）参数很少，用新数据训练即可。
- 最终输出仍是未来14天电压。

#### 与原模型代码的集成

在您的 `model.py` 中增加一个新类 `ConditionalVoltagePredictor`：

```python
class ConditionalVoltagePredictor(nn.Module):
    def __init__(self, base_model, future_action_dim=2, future_len=14, cond_hidden=32):
        super().__init__()
        self.base_model = base_model  # 加载原LSTM模型，可冻结或微调
        # 条件编码器：处理未来14天×2个动作
        self.cond_encoder = nn.Sequential(
            nn.Conv1d(future_action_dim, 16, kernel_size=3, padding=1),
            nn.ReLU(),
            nn.AdaptiveAvgPool1d(1),
            nn.Flatten(),
            nn.Linear(16, cond_hidden)
        )
        # 合并后的预测头
        self.fc = nn.Sequential(
            nn.Linear(128 + cond_hidden, 64),
            nn.ReLU(),
            nn.Linear(64, future_len)  # 输出14个电压值
        )
  
    def forward(self, past_features, future_actions, pot_ids):
        # past_features: (batch,7,84)
        # future_actions: (batch,14,2)
        hidden = self.base_model.get_hidden(past_features, pot_ids)  # (batch,128)
        cond = self.cond_encoder(future_actions.permute(0,2,1))  # (batch,cond_hidden)
        combined = torch.cat([hidden, cond], dim=1)
        return self.fc(combined)  # (batch,14)
```

#### 优势

- ✅ **原模型训练成果完全保留**，无需重新训练84维输入的特征提取能力。
- ✅ 只需额外训练轻量级条件编码器（参数量约几千），数据需求小。
- ✅ 未来动作作为条件输入，符合语义：模型回答“给定历史，如果未来这样做，电压会怎样”。
- ✅ 便于后续与MPD-PPO集成：仿真环境中，MPD-PPO输出未来动作序列（或单步动作，堆叠成序列），输入该模型即可得到预测电压。

---

### 四、总结

| 方案                                       | 原模型权重可用性       | 实施难度 | 推荐度               |
| ------------------------------------------ | ---------------------- | -------- | -------------------- |
| 直接修改输入维度，重新训练                 | 完全无用（需从头训练） | 中       | ❌ 不推荐            |
| 加载部分权重（LSTM层），随机初始化输入投影 | 部分可用               | 中       | ⚠️ 可行但非最优    |
| **保持原模型，增加条件编码器**       | **完全可用**     | 低       | ✅**强烈推荐** |

**建议**：采用第三种方案，只需在现有模型外部封装一个条件模块，即可实现“条件电压预测器”，且原模型的所有训练成果完全保留。这样您无需重新训练庞大的LSTM，只需用少量数据训练新增的小网络，即可快速支持MPD-PPO的仿真环境。
