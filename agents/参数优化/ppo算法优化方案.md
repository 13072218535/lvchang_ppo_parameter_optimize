## 适配现有电压预测模型的MPD-PPO优化方案

您的现有工作电压预测模型（LSTM + 槽嵌入）已经能够基于**前7天数据**预测**未来14天的工作平均电压**，但在直接嵌入MPD-PPO优化框架时存在一个关键缺口：**该模型输入中不包含“未来动作”**，因此无法回答“如果我明天改变ALF加料量和出铝量，未来电压会如何变化”。为了解决这个问题，同时**最大化复用您已有的代码和数据流程**，下面给出一个**直接适配**的方案，只需对现有模型进行最小化扩展，即可与MPD-PPO无缝集成。

---

### 一、核心思路：将预测模型改造为“条件电压预测器”

#### 1.1 现有模型输入输出回顾

- **输入**：过去7天窗口，每个时间步包含约84个特征（12个基础特征 + 统计/差分/衍生特征，已含历史 `ALF加料量(实际)`和 `实际出铝量`）。
- **输出**：未来14天的工作平均电压（14个连续值）。

#### 1.2 需要增加的“条件输入”

为了让模型能预测“假如我采取某组动作，电压会变成多少”，我们需要将**未来14天的动作序列**也作为输入特征。即新模型：

- **输入**：过去7天特征（不变） + 未来14天的两个可控变量序列：`ALF加料量(实际)` 和 `实际出铝量`。
- **输出**：未来14天的工作平均电压（与原来一样）。

#### 1.3 如何利用现有代码改造

您现有的 `data_processor.py`已经完成了特征工程，可以**无缝扩展**：

- 对于每个训练样本（取历史窗口起点 `t`），除原有的过去7天特征外，还需提取：
  - 未来14天（即 `t+1`到 `t+14`）的 `ALF加料量(实际)`和 `实际出铝量`真实历史值（训练时使用历史真实值，模拟环境时使用策略生成值）。
- 将这两个未来动作序列（2×14=28个额外数值）拼接到每个样本的输入特征中。注意这些动作在时间维度上**不是时间序列**（它们是未来计划），但为了让LSTM处理，可以将其作为额外的**静态特征**，或者扩展为与时间步对齐的序列。推荐方式：将未来14天的动作值复制成与时间步对齐的序列，在每个未来时间步上附加对应的动作值。

具体在LSTM中的处理：
输入张量形状变为 `(batch, past_len=7, past_feat_dim + future_action_dim_per_step)`，其中 `future_action_dim_per_step = 2`（每天两个动作）。对于过去7天，未来动作值取**未来对应天的动作**（即第t+1天的动作放在过去窗口的第1天位置，依此类推），超出未来14天范围的部分用0或中值填充。这样模型可以学习“过去工况 + 未来动作计划 → 未来电压”的映射。

#### 1.4 训练条件预测模型

- 使用相同的数据划分（按槽号），构造新的训练集、验证集。
- 模型结构完全复用您现有的 `LSTMModelWithPotEmbedding`，仅需修改输入维度 `input_dim` 从84增加到 `84 + 28`（未来14天×2个动作）。输出维度仍是14。
- 损失函数、训练参数均保持不变。
- 训练完成后，该模型即可作为**电压仿真环境**的核心，接受任意未来动作序列并预测电压响应。

### 二、MPD-PPO仿真环境设计（完全适配上述模型）

#### 2.1 环境定义（MDP）

| 组件                   | 设计                                                                                                                                                                                                    |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **状态** \(s_t\) | (1) 过去7天的所有特征（84维），(2) 未来14天的目标设定电压序列（14个值），(3) 槽号（用于嵌入）。共约84+14+1（槽号索引）。                                                                                |
| **动作** \(a_t\) | 二维连续：\(a_1\) = 明天 `ALF加料量(实际)`，\(a_2\) = 明天 `实际出铝量`。范围由历史数据5%~95%分位数确定。                                                                                           |
| **转移函数**     | 给定状态\(s_t\) 和动作 \(a_t\)，需要生成下一状态 \(s_{t+1}\)。我们利用条件预测模型预测未来14天电压，然后**只取第一天的预测电压**作为动作执行后的结果，然后滑动时间窗口一天，构造新的过去7天状态。 |
| **奖励** \(r_t\) | 基于预测电压与目标设定电压的误差，加上动作平滑惩罚、边界惩罚（见下文）。                                                                                                                                |
| **终止**         | 达到14步（预测窗口结束时）或累计误差过大（如>0.3V）提前终止。                                                                                                                                           |

#### 2.2 状态转移的具体计算

假设当前时间步为 `t`，已知：

- 过去7天真实特征矩阵 \(X_{t-6:t}\)（包含历史动作、电压等）。
- 未来14天目标电压序列 \(V_{target}(t+1...t+14)\)。
- 当前MPD-PPO输出的动作 \(a_t = [alf_t, out_t]\)。

**步骤1**：构建条件预测模型的输入：

- 过去7天特征：直接使用 \(X_{t-6:t}\)。
- 未来14天动作：需要给出 \(a_{t+1}...a_{t+14}\)。由于我们只决策了明天的动作，后面的动作尚不知晓，一种简单且有效的方法是：**假设未来动作保持为最近的历史值或工艺推荐值**（例如使用最近7天动作的中位数）。我们也可以让MPD-PPO输出整个序列，但会增加复杂度。为简化，采用**保持策略**：对于 \(t+2\) 到 \(t+14\) 的动作，使用前一天动作的重复或历史平均值。这样模型仍能给出合理电压预测。

**步骤2**：调用条件预测模型，得到未来14天电压预测 \(\hat{V}_{t+1...t+14}\)。

**步骤3**：计算奖励 \(r_t\)，并提取 \( \hat{V}_{t+1} \) 作为这一天执行动作后的电压。

**步骤4**：构造下一个状态 \(s_{t+1}\)：

- 新的过去7天窗口：丢弃最老的一天，加入新的一天。新的一天的特征中，`ALF加料量(实际)`和 `实际出铝量`设为 \(a_t\)（执行值），`工作平均电压`设为 \(\hat{V}_{t+1}\)，其他特征（如铝水平、电解质水平、Fe/Si等）如何更新？由于我们没有这些变量的动态模型，一个可行的近似是：**假设这些变量在一天内变化很小，沿用最新一天的值**。更精确的做法是建立完整的状态转移模型，但初期可以接受近似。因为在奖励中主要惩罚电压偏差，动作对其他变量的影响会间接体现在未来的电压预测中（因为条件预测模型输入包含这些变量，它们会随滑动窗口更新）。

**步骤5**：时间步 \(t \leftarrow t+1\)，重复直到14步结束。

#### 2.3 奖励函数（借鉴论文复合奖励）

定义误差 \(e_t = |\hat{V}_{t+1} - V_{target}(t+1)|\)（单步误差）。

\[
\begin{aligned}
R_{acc} &= 2.0 \cdot \exp(-e_t) \quad &\text{(精度奖励)}\\
R_{prog} &= 0.3 \cdot \tanh(e_{best} - e_t) \quad &\text{(进度奖励)}\\
P_{smooth} &= -0.05 \cdot \left[ (a_1^t - a_1^{t-1})^2 + (a_2^t - a_2^{t-1})^2 \right] \quad &\text{(动作平滑惩罚)}\\
P_{bound} &= \begin{cases} -10 & \text{if } a \notin [min,max] \\ 0 & \text{otherwise} \end{cases} \quad &\text{(边界惩罚)}\\
R_t &= R_{acc} + R_{prog} + P_{smooth} + P_{bound}
\end{aligned}
\]

其中 \(e_{best}\) 为当前episode内最小的单步误差。

### 三、MPD-PPO算法实现与适配

#### 3.1 网络结构（多分支策略）

直接采用论文中的设计，与您的现有模型**解耦**（独立训练）：

- **Actor网络**：共享底层全连接层（256+128），然后分两个独立分支，分别输出 \(a_1\) 和 \(a_2\) 的均值 \(\mu_i\) 和标准差 \(\sigma_i\)（softplus确保正数）。动作通过高斯采样得到。
- **Critic网络**：独立网络，输出状态价值 \(V(s)\)。
- **差异化裁剪阈值**：
  - ALF加料量（敏感，小步调整）：\(\epsilon_1 = 0.1\)（即裁剪范围 [0.9, 1.1]）
  - 实际出铝量（可稍大步）：\(\epsilon_2 = 0.2\)（裁剪范围 [0.8, 1.2]）

#### 3.2 训练流程（完全基于仿真环境）

您不需要在实际产线上试错，只需用历史数据训练好条件预测模型，然后在该模型模拟的环境中训练MPD-PPO。

**算法步骤**（每轮迭代）：

1. 从历史数据中随机选取一个起始时间点 \(t_0\)（不同槽号、不同日期），初始化环境状态（过去7天真实特征 + 未来目标电压序列）。
2. 运行一个episode（最多14步），每步：
   - 根据当前状态 \(s_t\)，Actor网络输出动作 \(a_t\)。
   - 环境执行动作，调用条件预测模型得到 \(\hat{V}_{t+1}\)，计算奖励，更新状态。
   - 存储 \((s_t, a_t, r_t, s_{t+1})\)。
3. 收集若干episodes后（例如2048个时间步），计算优势函数（GAE），更新Actor和Critic（使用PPO clip loss，多分支加权求和）。
4. 重复直到策略收敛（通常是电压误差MAE低于0.02V）。

#### 3.3 与您现有代码的集成点

| 现有模块              | 如何复用                                                                                                       |
| --------------------- | -------------------------------------------------------------------------------------------------------------- |
| `data_processor.py` | 用于加载原始数据、特征工程、标准化。改造后新增提取“未来14天动作”的功能，生成条件预测模型的训练样本。         |
| `model.py`          | 直接复用 `LSTMModelWithPotEmbedding`，仅修改 `input_dim`。训练条件预测模型的脚本可基于 `train.py` 修改。 |
| `predict.py`        | 可作为仿真环境中调用条件预测模型的接口。                                                                       |
| 槽嵌入                | 保留了电解槽个体差异信息，对提高预测精度很重要，建议继续使用。                                                 |

### 四、推荐值生成（实际部署流程）

当MPD-PPO训练完成后，每天为每台电解槽生成如下推荐值：

**输入**（实时）：

- 过去7天的真实数据（包括所有特征）。
- 未来14天的目标设定电压序列（由生产计划给定）。

**步骤**：

1. 使用训练好的MPD-PPO策略网络，输入当前状态 \(s_t\)，得到动作 \(a_t = [alf\_rec, out\_rec]\)。
2. 可选：调用条件预测模型，输入当前状态 + 未来动作（明天用推荐动作，后续天用保持策略），预测未来14天电压曲线，验证是否合理。
3. 输出推荐值：
   ```
   槽号 1101 明天推荐：
   ALF加料量(实际) = 32.5 kg
   实际出铝量 = 3860 kg
   预期明天工作电压 = 3.98 V (目标 3.97 V)
   未来7天电压趋势预测：...
   ```
4. 人工审核后执行（或直接下发给自动控制系统）。
5. 第二天收集真实电压和工况，更新数据库，并**定期（如每周）重新训练条件预测模型和MPD-PPO**，以适应电解槽老化等时变特性。

### 五、实施路线图与注意事项

| 阶段 | 任务                                                                         | 工作量 | 依赖                       |
| ---- | ---------------------------------------------------------------------------- | ------ | -------------------------- |
| 1    | 扩展数据处理器，生成条件预测训练样本（过去7天+未来14天动作 → 未来14天电压） | 1天    | 现有 `data_processor.py` |
| 2    | 训练条件预测模型（复用 `model.py` 和 `train.py`）                        | 1-2天  | 阶段1                      |
| 3    | 实现仿真环境类（封装条件预测模型，实现MDP转移）                              | 2天    | 阶段2                      |
| 4    | 实现MPD-PPO算法（多分支Actor、Critic、差异化裁剪）                           | 3-4天  | 阶段3                      |
| 5    | 在仿真环境中训练MPD-PPO，调优奖励权重                                        | 2-3天  | 阶段4                      |
| 6    | 生产部署：编写每日推荐脚本，对接实时数据库                                   | 1-2天  | 阶段5                      |
| 7    | 试点运行（1-2台槽，开环建议）                                                | 1周    | 阶段6                      |

**关键注意事项**：

- **条件预测模型的精度**：如果未来14天电压预测的MAE > 0.03V，强化学习策略可能不可靠。建议先验收预测模型（要求MAE < 0.02V），再训练MPD-PPO。
- **动作平滑约束**：实际生产中，ALF加料量日变化不宜超过10%，出铝量日变化不宜超过100kg。在奖励函数中加重平滑惩罚，并在策略输出后clip到合理变化率。
- **槽嵌入复用**：条件预测模型中已经学习了每个槽号的偏置特性，MPD-PPO的状态中包含槽号，因此不同槽号可以共享同一个策略网络（通过槽号特征自动适应）。
- **冷启动**：新开槽（槽龄<30天）没有足够历史数据，可暂用同型槽的平均策略或人工经验。
