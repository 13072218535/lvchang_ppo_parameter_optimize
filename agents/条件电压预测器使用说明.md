# 条件电压预测器使用说明

## 概述

本文档说明如何使用改造后的"条件电压预测器"来预测给定未来动作序列下的电压变化。

## 主要修改

### 1. 数据处理 (data_processor.py)

- **VoltageDataset类**：新增`future_actions`参数，用于存储未来14天的动作序列
- **create_sequences方法**：新增`use_future_actions`参数，控制是否提取未来动作序列
- **process方法**：新增`use_future_actions`参数，控制整个数据处理流程

### 2. 模型结构 (model.py)

- **LSTMModelWithPotEmbedding类**：新增`get_hidden`方法，用于获取LSTM的隐藏状态
- **ConditionalVoltagePredictor类**：新增条件预测器，包含：
  - 基础LSTM模型（复用原模型）
  - 条件编码器（Conv1D + 全连接层）
  - 合并预测头

### 3. 训练脚本 (train.py)

- **train_epoch函数**：新增`use_conditional`参数，支持条件预测模型训练
- **validate_epoch函数**：新增`use_conditional`参数，支持条件预测模型验证
- **test_evaluate函数**：新增`use_conditional`参数，支持条件预测模型测试
- **main函数**：新增`use_conditional`配置项，控制是否使用条件预测模型

### 4. 配置文件 (config.py)

- **OUTPUT_LEN**：从30改为14，预测未来14天电压

## 使用方法

### 方法1：训练基础模型（原有功能）

```python
# 在train.py的main函数中设置
use_conditional = False  # 使用原有的LSTM模型
```

然后运行：
```bash
python train.py
```

### 方法2：训练条件预测模型（新功能）

```python
# 在train.py的main函数中设置
use_conditional = True  # 使用条件预测模型
```

然后运行：
```bash
python train.py
```

**注意**：如果存在预训练的基础模型权重（`output/best_model.pth`），会自动加载并冻结基础模型参数，只训练条件编码器和预测头。

## 条件预测模型架构

```
输入:
- 过去7天特征: (batch_size, 7, 84)
- 未来14天动作: (batch_size, 14, 2)  [ALF加料量, 实际出铝量]
- 槽号索引: (batch_size,)

处理流程:
1. 基础LSTM模型处理过去7天特征 → 隐藏状态 (batch_size, 128)
2. 条件编码器处理未来14天动作 → 条件向量 (batch_size, 32)
3. 拼接隐藏状态和条件向量 → 合并特征 (batch_size, 160)
4. 预测头处理合并特征 → 未来14天电压 (batch_size, 14)
```

## 迁移学习策略

根据`模型加载迁移学习.md`文档，采用"保持原模型，增加条件编码器"的策略：

1. **完全复用原模型权重**：基础LSTM模型的所有参数都可以从预训练模型加载
2. **冻结基础模型**：加载预训练权重后，冻结基础模型参数，避免破坏已学到的特征
3. **只训练新增部分**：只训练条件编码器和预测头，参数量少，训练速度快

## 与MPD-PPO的集成

条件预测模型可以作为MPD-PPO仿真环境的核心组件：

1. **状态**：过去7天特征 + 未来14天目标电压 + 槽号
2. **动作**：明天的ALF加料量和实际出铝量
3. **环境转移**：
   - 使用条件预测模型预测未来14天电压
   - 提取第一天的预测电压作为执行结果
   - 滑动时间窗口，构造新状态
4. **奖励**：基于预测电压与目标电压的误差

## 数据格式

### 输入数据

- **过去特征**：84维特征（12个基础特征 + 统计/差分/衍生特征）
- **未来动作**：2维动作（ALF加料量、实际出铝量），共14天
- **槽号**：槽号索引（0到num_pots-1）

### 输出数据

- **未来电压**：14个工作平均电压预测值

## 测试

运行测试脚本验证模型功能：

```bash
python test_conditional_model.py
```

该脚本会测试：
1. 基础LSTM模型创建
2. 条件预测器创建
3. 前向传播
4. 隐藏状态提取
5. 条件编码器
6. 梯度传播

## 注意事项

1. **数据要求**：每个槽号至少需要21天数据（7天输入 + 14天输出）
2. **动作特征**：确保数据中包含`ALF加料量(实际)`和`实际出铝量`列
3. **预训练模型**：如果使用迁移学习，确保`output/best_model.pth`存在
4. **输出长度**：OUTPUT_LEN已改为14，与方案要求一致
