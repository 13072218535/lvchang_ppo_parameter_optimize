### 1.2 奖励函数参数调优

针对三个问题（Reward起始过高、后期发散、P_bound恒为0）进行参数优化：

| 参数 | 修改前 | 修改后 | 效果 |
|------|--------|--------|------|
| `REWARD_ACC_KAPPA` | 2.0 | **15.0** | 0.1V vs 0.3V 误差区分度从1.15x→3.22x |
| `REWARD_ACC_WEIGHT` | 10.0 | **8.0** | 配合κ调整精度权重占比 |
| `REWARD_SMOOTH_VIOLATION_WEIGHT` | 5.0 | **20.0** | 强化平滑约束，防止牺牲平滑换精度 |
| `REWARD_SMOOTH_EXTREME_MULTIPLIER` | — | **3.0** (新增) | \|Δ\|>2×上限时惩罚×3，阻止极端违规 |
| `REWARD_BOUND_MARGIN` | — | **0.10** (新增) | 边界10%区域触发接近惩罚(替代无效的超界检查) |

**P_bound修复**：将原来"动作≥max才惩罚"（因tanh→(-1,1)映射到开区间永不触发）改为"进入[max-margin, max]区域即线性惩罚"，使边界约束真正生效。

### 2.3 算法架构设计

| 组件 | On-policy（PPO/A2C） | Off-policy（DDPG/TD3/SAC） |
|------|---------------------|--------------------------|
| **Actor** | LSTM编码器 → 28维mean+std | LSTM编码器 → **4维**(start,end) → TrajectoryExpander → 28维 |
| **Critic** | LSTM V-Critic V(s) | **MLP Q-Critic** Q(s,a)（10-50x更快） |
| **经验利用** | 无回放，逐批GAE更新 | ReplayBuffer(100K)，每步更新 |
| **探索策略** | 学习到的对角高斯std | **AR(1)时序相关噪声**（ρ=0.85，保持日间平滑） |
| **更新频率** | 256步/批次 | 每步更新（TD3每2步更Actor） |

## 三、6算法200轮对比实验结果

### 3.1 初始对比（修复前）

| 算法 | 类型 | 最终Reward | 训练耗时 | 结果 |
|------|------|:---:|:---:|:---:|
| **MPD-PPO** | On-policy | 732.81 | 1194s | ✅ 收敛 |
| **Vanilla PPO** | On-policy | 736.27 | 1145s | ✅ 收敛 |
| **A2C** | On-policy | 736.11 | 867s | ✅ 收敛 |
| **DDPG** | Off-policy | -11,461 | 2590s | ❌ 发散 |
| **TD3** | Off-policy | -10,055 | 2594s | ❌ 发散 |
| **SAC** | Off-policy | -12,743 | 3379s | ❌ 发散 |

### 3.2 Off-policy发散根因诊断

**违反1**：`self.last_action`不在状态中 → `r ≠ f(s,a)` → 贝尔曼方程不成立
**违反2**：`cumulative_error`不在状态中 → `done ≠ f(s,a)`
**违反3**：reward依赖全部28维动作，状态转移仅用day-1 → Q-learning信噪比极低
**违反4**：i.i.d.高斯噪声(28维) → 锯齿状轨迹 → P_smooth爆炸(-5000+)

## 四、环境马尔可夫性修复

### 4.1 状态空间扩展（99→102维）

```python
# 新增3维:
last_action (2维)  # 修复违反1: reward不再依赖隐藏变量
cumulative_error (1维)  # 修复违反2: done信号可预测
```

### 4.2 加权动作状态转移（用户方案）

```python
# 旧: alf_feature = action[0]  (仅day-1)
# 新: alf_feature = Σ(time_w[i] × future_actions[i, 0]) / Σ(time_w)
#  time_w = [1.0, 0.9, ..., 0.1] — day-1权重高, day-14权重低
```

使`s'`编码完整14天计划信息，Q(s',a')的信息视野与reward一致。

## 五、Off-policy三项专项优化

### 改进1：时序相关噪声（最关键突破）

```python
# 旧: i.i.d.高斯噪声 → P_smooth = -5000+
noise = torch.randn(28) * 0.2

# 新: AR(1)时序相关噪声 → P_smooth ≈ 0
noise[t] = ρ × noise[t-1] + √(1-ρ²) × ε[t]  # ρ=0.85
```

ALF和OUT各自生成14天AR(1)噪声，保持日间相关性。

### 改进2：4维动作空间缩减

```
Actor输出: 4维 [alf_start, alf_end, out_start, out_end]
    ↓ TrajectoryExpander（线性插值）
28维轨迹: [alf1, out1, ..., alf14, out14]
```

动作空间从28维降到4维，Q-learning泛化难度指数级降低。通过线性插值保证轨迹天然平滑。

### 改进3：课程学习

```python
# epoch 0:   smooth_weight = 1.0  (几乎无平滑惩罚，专注学习精度)
# epoch N×0.7: smooth_weight = 20.0 (完整惩罚，精细调优)
cur_weight = 1.0 + (REWARD_SMOOTH_VIOLATION_WEIGHT - 1.0) × progress
```

让Q函数先学会"什么是好动作"，再加平滑约束。

## 六、MPD-PPO专项改进

### 轨迹平滑正则化

在Actor loss中直接惩罚14天mean输出的日间变化：

```python
means_reshaped = means.view(-1, 14, 2)  # (batch, 14, 2)
day_diffs = means_reshaped[:, 1:, :] - means_reshaped[:, :-1, :]
smooth_reg = weights × (relu(|diff| - threshold))²
actor_loss += smooth_reg
```

**效果**：P_smooth从-947快速收敛到≈0（epoch 80+），比Vanilla PPO更高效地实现轨迹平滑。

### 两项尝试（已移除）

| 改进 | 问题 | 原因 |
|------|------|------|
| 时间加权Advantage | reward从+730→-12894 | 梯度失衡（day-1权重10×于day-14），撕裂轨迹 |
| 熵退火 | reward从+730→-12894 | 28维联合熵推高std，随机探索破坏平滑性 |

## 八、关键经验总结

1. **On-policy(PPO/A2C)是本环境的最优选择**：GAE天然处理非马尔可夫奖励，无需Q函数
2. **差异化裁剪(MPD-PPO的核心创新)在归一化动作空间中无实际收益**：ALF和OUT经tanh归一化后在概率空间中无区别
3. **Off-policy发散的根本原因是环境设计缺陷**（隐藏状态变量+28维→2维信息不对称），非超参数可修复
4. **三项改进(时序噪声+4维动作+课程学习)使off-policy从发散→收敛**：证明了架构适配比超参数调优更重要
5. **轨迹平滑正则化是MPD-PPO最有价值的改进**：直接在策略层面约束平滑性，比reward间接信号更高效
6. **推荐使用 Vanilla PPO 或 改进后的 MPD-PPO**：收敛快、稳定性好、轨迹平滑度高
