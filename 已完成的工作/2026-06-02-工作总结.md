# 2026-06-02 工作总结：仿真器不确定性量化 + 对抗训练

## 一、完成工作概览

本日围绕"提升仿真器保真度"这一核心目标，完成了从问题诊断到解决方案验证的完整闭环。

## 二、问题诊断

通过多轮实验确认：**RL 算法优化已接近天花板，瓶颈在仿真器（条件电压预测器）而非 RL 算法本身。**

| 发现 | 证据 |
|------|------|
| On-policy 算法 MAE 天花板 | MPD-PPO/Vanilla PPO/TAA-PPO/A2C 均稳定在 13.2 mV，无法超越 DDPG(12.8 mV) |
| Off-policy 结构优势 | DDPG/SAC 的 AR(1)噪声 + Replay Buffer 天然限制外推风险 |
| MLP Critic 是 TAA-PPO-4D 不收敛根因 | 67K MLP 无法可靠估计 V(s)，导致 Critic Loss 4000+、P_smooth=-500 |

## 三、解决方案

### 1. 仿真器不确定性量化（Ensemble UQ）

**新增文件：**
- `model/ensemble_predictor.py` — `UncertaintyQuantifiedPredictor` 类，持有 5 个条件预测器，推理时输出均值+方差
- `model/train_ensemble.py` — Ensemble 训练脚本（--phase base/conditional/calibrate）

**修改文件：**
- `ppo参数优化/model/config.py` — 新增 UNC_LAMBDA、UNC_USE_THRESHOLD 等配置
- `ppo参数优化/model/environment.py` — _predict_voltage 返回 (pred, uncertainty)；_calculate_reward 增加 P_unc 惩罚项；load_predictor_model 支持 ensemble 加载
- `ppo参数优化/model/train_compare.py` — 自动检测 ensemble checkpoints，传入 uncertainty_config

### 2. P_unc 惩罚项改进（加法 → 乘法折扣）

**问题**：λ=1 加法形式下 P_unc ≈ -0.01，相比 R_acc (~748) 完全可忽略。

**修改**：将 `R = R_acc + P_smooth + P_bound - λ·var` 改为 `R = R_acc × exp(-λ·var) + P_smooth + P_bound`，λ=50。

**效果**：P_unc 从 -0.01 变为 -65~-180（占 R_acc 9-25%），agent 被迫关注不确定性。

### 3. TAA-PPO-4D 专项优化

**3.1 P_bound 权重调整**：新增 `TAA4D_BOUND_PENALTY_WEIGHT=3.0`（原通用值 10），4D 线性插值无法避免边界接近。

**3.2 LSTM Critic 替换 MLP Critic**：将 `taa_ppo_4d.py` 中的 VNetworkMLP(67K) + CriticReplayBuffer 替换为 ppo.Critic(243K，LSTM+槽嵌入)。

**效果**：
- Critic Loss: 4000+ → 1.0
- P_smooth: -250~-700 → 0
- P_bound: -80~-710 → 0
- 最佳奖励: 329 → 694
- 奖励全正且收敛

### 4. 测试评估增强

在 `visualize_compare.py` 中增加人类基准对比：
- 每样本显示"Human: MAE=X mV"和"vs Human: +Y%"
- 总览图含人类基线参考线
- 汇总表新增"vs Human"列和 Win Rate 统计

### 5. 对抗训练（Adversarial Training）

**核心理念**：用 RL agent 探索的动作序列扩充预测器训练集，让预测器学会在 agent 探索区域也给出准确预测。

**新增文件：**
- `model/adversarial_augment.py` — 三阶段流水线
  1. 加载 8 个 RL agent → 在训练槽上 rollout → 收集动作序列
  2. 用 Ensemble 打伪标签 + 计算 uncertainty
  3. 筛选高不确定性样本 → 保存增强数据集

**修改文件：**
- `model/data_processor.py` — 新增 `load_augmented_sequences()` 方法，`process()` 支持 `augmented_data_path` 参数
- `model/train_ensemble.py` — 新增 `--use_augmented` 标志和 `augmented_data_path` 参数

**效果**（新旧预测器对比）：

| 算法 | 旧 MAE | 新 MAE | 改善 |
|------|--------|--------|------|
| SAC | 12.8 mV | **8.6 mV** | **-33%** |
| DDPG | 12.8 mV | **9.1 mV** | **-29%** |
| TAA-PPO-4D | 13.3 mV | **9.3 mV** | **-30%** |
| TD3 | 13.1 mV | **9.3 mV** | **-29%** |
| Vanilla PPO | 13.2 mV | **9.5 mV** | **-28%** |
| TAA-PPO | 13.2 mV | **9.5 mV** | **-28%** |
| MPD-PPO | 13.2 mV | **9.8 mV** | **-26%** |
| A2C | 13.7 mV | **10.0 mV** | **-27%** |
| **均值** | **13.2 mV** | **9.4 mV** | **-29%** |

全部算法平均改善 29%，单轮对抗训练即效果显著。

## 五、关键结论

1. **预测器保真度是最大瓶颈**：对抗训练使全部 RL 算法 MAE 下降 29%，远超任何 RL 算法层面的优化
2. **SAC 是最优算法**（8.6 mV，胜率 100%），但 TAA-PPO-4D 紧随（9.3 mV）
3. **LSTM Critic 是 on-policy 训练的必要条件**：MLP Critic 无法可靠估计 V(s)
4. **乘法 P_unc 公式正确有效**：量纲匹配，惩罚力度适中
5. **迭代对抗训练值得尝试**：单轮 29% 改善，多轮可能进一步缩小 on/off-policy 差距
