# 参数优化方案问题分析与改进建议

## 背景

本项目的任务流程是：根据电解槽过去7天数据 → 条件电压预测器预测未来14天工作电压 → 根据预测结果调整 ALF加料量(实际) 和 实际出铝量 → 使工作平均电压贴近设定电压(实际设定)。

当前实现了一个完整的 **模型预测(条件电压预测器) → PPO强化学习参数优化** 流水线，但存在若干关键问题需要改进。

---

## 一、核心问题（严重）

### 问题1：优化目标错位 — PPO被训练为匹配历史电压而非设定电压

**现状**：`environment.py` 的 `reset()` 中 `target_voltage` 来源于历史数据中的 `工作平均`（实际工作电压），`_calculate_reward()` 的误差计算为 `e_t = abs(voltage_pred[0] - target_voltage[0])`，即比较预测电压与历史实际电压的偏差。

**后果**：PPO Agent 学到的策略是"如何操作能让工作电压保持历史轨迹"，而不是"如何操作能让工作电压逼近设定电压"。如果在历史数据中，某槽的工作电压长期偏离设定值 0.05V，Agent 会被训练为维持这个偏差。

**改进方案**：将 `target_voltage` 改为未来14天的 `实际设定`（设定电压），奖励函数改为 `e_t = abs(voltage_pred[0] - set_voltage[0])`。改动涉及：
- `train_ppo.py` 的 `load_data_for_ppo()`：提取 `pot_data['实际设定'].values` 而非 `pot_data['工作平均'].values`
- `environment.py` 的 `_calculate_reward()`：保持计算公式不变，但 `target_voltage` 含义变为设定电压


### 问题2：未来动作的"保持策略"导致预测器输入失真

**现状**：`environment.py` 的 `_build_future_actions()` 中，将当前步的动作重复填充为全部14天的动作序列（全部14天用相同的ALF和出铝量）。

**后果**：
1. 条件预测器训练时看到的动作序列是逐日变化的，推理时却收到恒定序列——训练/推理分布不一致，预测准确性不可靠
2. PPO在第1步的预测假设"未来14天都用同一个动作A"，但第2步实际用了不同的动作B——多步轨迹在预测器视角下自相矛盾
3. 无法实现分阶段的轨迹规划（例如先逐步增加ALF、再维持、再递减）

**改进方案**：
- **方案A（低侵入）**：让Actor一次性输出14天×2个动作的完整计划(28维)，一次性喂给预测器，一次性获得14天电压预测和奖励。这样消除了单步交互与批量预测之间的矛盾。
- **方案B（中等侵入）**：保持14步交互，但每步`_build_future_actions()`使用当前策略自洽填充未来动作——即用Actor输出未来每一天的动作而非简单重复。


### 问题3：预测电压回灌导致误差累积

**现状**：`environment.py` 的 `_update_state()` 中，每步将 `voltage_pred[0]` 写入状态窗口的最新一天，然后重新计算所有统计特征（均值、标准差、差分等）。

**后果**：
1. 第1步的预测误差ε₁混入统计特征 → 第2步预测产生ε₂（包含ε₁的贡献）→ 误差级联放大
2. 步数超过7步后，状态窗口全部由预测值构成，预测器从未在训练中见过这种全合成数据
3. 这是模型驱动RL中的典型"分布偏移"问题

**改进方案**：
- 短期：考虑方案A（一次性输出14天计划），从根本上避免状态回灌
- 中期：如果保留步进式交互，可以给预测器增加不确定性估计（如MC Dropout），在状态更新时对低置信度预测进行保守处理
- 长期：在预测器训练数据中混入一定比例的"预测值作为输入"的数据增强（类似于Scheduled Sampling）


## 二、重要问题

### 问题4：Actor输入的目标电压将在部署时发生分布偏移

**现状**：`ppo.py` 的Actor网络中，`target_voltage`（14维）是Actor的核心条件输入，与LSTM隐藏状态和槽嵌入拼接后驱动动作决策。训练时这个向量是历史的实际工作电压（含噪声和波动），部署时却被替换为设定电压（通常是平稳的常数值）。

**后果**：Actor从未见过"干净的常数值设定电压"作为输入，其网络权重对这种输入的行为不可预测。

**改进方案**：配合问题1的修复，训练时就使用设定电压作为 `target_voltage`，使训练-部署一致。


### 问题5：PPO步进式交互与预测器批量预测之间的结构矛盾

**现状**：PPO假设标准MDP：`s_t → a_t → r_t, s_{t+1}`。预测器却设计为离线批量预测：`(过去7天, 14天动作计划) → 14天电压预测`。二者通过"保持策略"桥接，造成了结构不一致。

**后果**：这个结构矛盾是问题2和问题3的根源。每次单步交互时，预测器被迫接受一个不真实的全14天动作序列，同时只有第1天的预测被用于奖励计算，剩余的13天预测被丢弃。

**改进方案**：
- **推荐方案**：改用**模型预测控制(MPC)**范式替代PPO。将预测器视为可微分世界模型，直接对完整14天动作序列做梯度下降优化：
  ```
  min_{a_1..a_14}  Σ (predicted_voltage_i - set_voltage)^2 + λ * smoothness_penalty
  ```
  每次优化只做几十步梯度下降，无需训练独立的Actor-Critic网络，无需PPO的繁琐经验收集。预测器本身就是可微的（PyTorch模型），梯度可直接回传至动作变量。


### 问题6：非控制特征在仿真期间被冻结

**现状**：`_update_state()` 仅更新了3个特征（工作平均、ALF加料量(实际)、实际出铝量），其余80+个特征（铝水平、电解质水平、Fe含量、电流效率、平均电压等）均被冻结为最后一天的值。

**后果**：如果优化器建议大幅改变ALF加料量，在真实环境中这会影响电解质成分、进而影响Fe含量和电压，但仿真中这些连锁反应被忽略。仿真与真实物理过程的差距增大。

**改进方案**：
- 短期：至少对与ALF和出铝量有已知物理关系的特征（如铝水平、电解质水平、平均电压）建立简单的经验更新规则
- 中期：训练辅助子模型预测这些特征的演变


## 三、中等问题

### 问题7：差异化裁剪的乘积实现引入了不必要的耦合

**现状**：`ppo.py` 中 `clipped_ratio = clipped_alf_ratio * clipped_out_ratio`，将两个动作维度的裁剪比率相乘作为整体裁剪比率。

**后果**：ALF的裁剪比率[0.9, 1.1]乘以OUT的[0.8, 1.2]，有效范围为[0.72, 1.32]——实际裁剪范围与设计意图偏移。同时两个动作头之间存在乘性梯度耦合，一个头的ratio波动会影响另一个头的更新幅度。

**改进方案**：改为标准的独立裁剪+求和方法：分别计算两个动作头的clipped surrogate loss，然后相加。


### 问题8：奖励仅使用第1天预测，浪费了预测器的多步输出

**现状**：预测器输出完整的14天电压，但奖励 `R_acc = 10 * exp(-2 * e_t²)` 中 `e_t` 仅用 `voltage_pred[0]`。

**后果**：Agent不关心未来13天的电压质量。如果某动作第1天效果不错但会导致后续电压恶化，Agent会继续选择它。

**改进方案**：将奖励扩展为多步加权，例如 `R_acc = Σ_{i=0}^{13} w_i * exp(-κ * (voltage_pred[i] - set_voltage[i])²)`，权重 `w_i` 随时间衰减。


### 问题9：批次大小配置与实际不一致

**现状**：`PPO_BATCH_SIZE = 64`，但每episode精确14步，实际收集到 5×14=70 步才停止。

**后果**：不影响正确性，但 `PPO_BATCH_SIZE` 命名有误导性。同时缺少PPO标准的内层mini-batch切分。


### 问题10：确定性预测器无不确定性度量

**现状**：预测器是确定性模型，给出精确预测值。PPO完全信任预测结果。

**后果**：Agent可能利用预测器的系统性偏差，学到一个在仿真中表现好但在现实中无效的策略。

**改进方案**：在预测器训练时使用MC Dropout或Ensemble方法，推理时给出预测均值+方差，用于奖励函数中做不确定性折价。


## 四、改进优先级建议

| 优先级 | 改进项 | 理由 |
|--------|--------|------|
| P0 | 问题1+4：将优化目标从历史电压改为设定电压 | 不修复则整个优化方向错误 |
| P0 | 问题5+2+3：将PPO步进交互改为MPC一次性轨迹优化 | 从根源消除结构矛盾、误差累积、动作序列失真三个问题 |
| P1 | 问题8：扩展奖励到多步加权 | 让优化器考虑完整14天轨迹质量 |
| P1 | 问题6：改进非控制特征的更新逻辑 | 增加仿真物理合理性 |
| P2 | 问题7：修复clipped_ratio乘积耦合 | 提升PPO更新稳定性（如果保留PPO方案） |
| P2 | 问题9：修正批次大小，增加mini-batch | 提升PPO训练质量（如果保留PPO方案） |
| P3 | 问题10：预测器不确定性量化 | 提升策略鲁棒性 |

## 五、推荐技术路线

**首选方案：MPC（模型预测控制）替代PPO强化学习**

```
输入：过去7天特征 + 未来14天设定电压 + 槽号
过程：
  1. 初始化14天动作序列 a = [a_1, a_2, ..., a_14]（可从历史均值开始）
  2. 将 a 输入条件预测器，得到14天电压预测
  3. 计算损失 L = MSE(预测电压, 设定电压) + smoothness_penalty(a)
  4. 对 a 做梯度下降（预测器冻结），更新动作序列
  5. 重复2-4直到收敛（通常20-50步梯度下降）
输出：优化后的14天动作序列 a*
```

**理由**：
- 预测器本身就是可微模型，梯度可直达动作变量——天然的端到端优化
- 无需训练Actor-Critic网络，无需管理经验缓存和GAE计算
- 一次性输出完整14天轨迹，消除了步进交互的所有矛盾
- 优化速度远快于PPO（每次推理几十步梯度下降 vs. 上千轮PPO训练）
- 对每个槽、每个时刻都可以独立运行优化，无需"训练-部署"两阶段
