# 铝电解槽工作电压参数优化项目 — 代码分析总览

## 一、项目概览

本项目针对铝电解槽的**工作电压控制优化**问题，构建了一套完整的"**条件电压预测 + PPO强化学习优化**"流水线。核心目标：通过智能调整 `ALF加料量` 和 `实际出铝量` 两个可控参数，使电解槽的**工作平均电压尽可能逼近设定电压**，从而提升电流效率和降低能耗。

项目已按《PPO参数优化改进方案》完成一轮重构，将原"单步交互PPO"升级为"滚动时域轨迹PPO"（Trajectory-Aware PPO），动作空间从2维扩展到28维（14天×2动作），消除了训练/推理分布不一致的根本缺陷。

### 3.1 `model/` — 条件电压预测模型

这是整个系统的**基础层**，负责回答核心问题："给定过去7天工况 + 未来14天操作计划 → 未来14天电压会是多少？"

#### 3.1.1 `model/config.py` — 预测模型配置

定义预测模型的所有超参数和数据处理参数：

| 参数类别 | 关键参数 | 说明 |
|---------|---------|------|
| 序列配置 | `INPUT_LEN=7`, `OUTPUT_LEN=14` | 输入7天，预测14天 |
| 模型结构 | `HIDDEN_DIM=128`, `NUM_LAYERS=2`, `POT_EMBED_DIM=16` | 2层LSTM + 槽号16维嵌入 |
| 训练控制 | `BATCH_SIZE=64`, `NUM_EPOCHS=100`, `LEARNING_RATE=0.001` | Adam优化器，ReduceLROnPlateau调度 |
| 早停 | `EARLY_STOPPING_PATIENCE=15` | 验证集loss不降15轮停止 |
| 数据划分 | `TRAIN_POTS`(30个槽), `VAL_POTS`(6个), `TEST_POTS`(6个) | 按槽号独立划分，避免数据泄露 |
| 特征选择 | `HIGH_CORR_FEATURES`(12个) | 基于Spearman相关系数筛选（\|r\|≥0.2） |

#### 3.1.2 `model/data_processor.py` — 数据处理流水线

**类 `VoltageDataset`**：PyTorch Dataset封装，支持四种样本格式（纯特征/带槽号/带未来动作/完整）。

**类 `DataProcessor`**：完整的数据处理流水线，6步流程：
1. `load_data()` — 读取Excel，按日期排序
2. `preprocess_data()` — 缺失值处理（分组前向填充 + 线性插值 + 均值填充）
3. `create_features()` — 特征工程：
   - 为每个高相关性特征创建：3天/7天滑动统计（mean/std）、1阶/7阶差分
   - 衍生特征：电压偏差、铝电解比例、槽龄_log、槽龄_squared
   - 最终特征维度约 **84维**（12基础 + 60统计/差分 + 4衍生 + 8目标变量统计）
4. 数据划分 — 支持两种模式：
   - `time`：每个槽号内按时间6:2:2划分
   - `pot`：按槽号分组划分（训练30/验证6/测试6）
5. 标准化 — 仅在训练集上fit scaler，然后transform全部
6. 序列构建 — 滑动窗口切分(7天输入 + 14天输出)

**特别设计**：`use_future_actions=True` 时额外提取未来14天的 `[ALF加料量(实际), 实际出铝量]` 动作序列，形状为 `(样本数, 14, 2)`，用于训练条件预测模型。

#### 3.1.3 `model/model.py` — 神经网络定义（三模型层次）

| 类名 | 层级 | 输入 | 输出 | 说明 |
|------|------|------|------|------|
| `LSTMModel` | 基础 | `(B,7,feat_dim)` | `(B,14)` | 纯LSTM，无槽嵌入 |
| `LSTMModelWithPotEmbedding` | 增强 | `(B,7,feat_dim)` + `(B,)` pot_ids | `(B,14)` | LSTM + 槽号16维嵌入 + BatchNorm |
| `ConditionalVoltagePredictor` | 条件 | `(B,7,feat_dim)` + `(B,14,2)` 动作 + pot_ids | `(B,14)` | 拼接"历史隐藏状态"和"条件动作编码"后预测 |

**`ConditionalVoltagePredictor` 架构**（关键设计）：

```
过去7天特征 → [冻结的]LSTM+槽嵌入 → hidden(128维) ──┐
                                                        ├→ concat(160维) → FC(64→32→14) → 未来14天电压
未来14天动作 → Conv1D→AvgPool→FC(32维) ──────────────┘
```

采用迁移学习：基础LSTM加载预训练权重后冻结，仅训练条件编码器和预测头。这保证了已学到的时序特征不被破坏。

#### 3.1.4 `model/train.py` — 预测模型训练

完整的训练流水线：

- **评估函数**：`calculate_metrics()` (MAE/MSE/RMSE/MAPE/R²)、`evaluate_by_day()` (逐日评估)
- **训练引擎**：`train_epoch()` / `validate_epoch()` — 适配三种模型类型（基础/槽嵌入/条件）
- **可视化**：6种图表 — 损失曲线、分日指标、样本对比、全量散点、全量时序、组合图
- **模型保存**：最佳模型(`best_model.pth`) + 条件最佳(`best_conditional_model.pth`) + 最终模型
- **结果输出**：`test_results.txt` 包含完整评估指标和特征列表

#### 3.1.5 `model/predict.py` — 预测推理

命令行推理工具，支持两种模式：
- **文件模式**：读取CSV → 取最近7天 → 预测未来14天电压 → 保存CSV
- **演示模式**：使用模拟数据演示预测能力

| **PPO训练** | `PPO_STEPS_PER_UPDATE=256` | 每次更新前收集256步经验 |
| | `PPO_MINI_BATCH_SIZE=64` | 内层mini-batch大小 |

#### 3.2.2 `ppo参数优化/model/model.py` — 预测模型定义

与 `model/model.py` 内容完全一致（同源代码），供PPO模块独立导入使用，不依赖模块A的路径。

#### 3.2.3 `ppo参数优化/model/environment.py` — 电压控制仿真环境（MDP核心）

**类 `VoltageControlEnv`** — 封装条件预测模型，实现标准的强化学习环境接口：

**状态空间** (`state_dim = 7×input_dim + 14 + 1`):
- 过去7天完整特征窗口（展平）
- 未来14天目标设定电压序列
- 槽号索引

**动作空间** (`action_dim = 28`):
- 14天×2动作的完整轨迹 `[alf₁, out₁, alf₂, out₂, ..., alf₁₄, out₁₄]`

**核心方法**：

| 方法 | 功能 |
|------|------|
| `reset(past_features, target_voltage, pot_id)` | 初始化状态，返回s₀ |
| `step(action_trajectory_28d)` | 执行28维轨迹，返回 (s', r, done, info) |
| `_reshape_trajectory(action_28d)` | 28维→(14,2)重塑 |
| `_predict_voltage(...)` | 调用条件预测模型 |
| `_calculate_reward(...)` | 多步加权奖励计算 |
| `_update_state(action, voltage_pred, prev_features)` | 滑动窗口 + 经验更新 |
| `_update_statistical_features(features)` | 重新计算统计特征 |

**状态转移流程**（滚动时域）：

```
输入28维轨迹 → 重塑为(14,2) → 反归一化+裁剪 → 预测14天电压
→ 计算14天加权奖励 → 仅执行day-1动作 → 滑动窗口1天
→ 经验更新非控特征 → 重算统计特征 → target用末值填充保持14维 → 返回s_{t+1}
```

注：target_voltage 滑动后用末值填充而非缩减（修复形状崩溃 bug）。

**`_update_state()` 中的经验更新规则**：
- **工作平均电压**：用预测值 voltage_pred[0] 更新
- **ALF加料量/出铝量**：用 day-1 动作更新
- **铝水平**：受出铝量变化影响（经验公式 `-0.3 × (Δout/prev_out) × aluminum_level`）
- **平均电压**：直接用预测值
- **电压设定/实际设定**：沿用前一天值（缓慢变化假设）
- **统计特征**：基于新窗口重新计算均值/标准差/差分

**`_calculate_reward()` — 多步加权奖励函数**（已修复）：

```
R = R_acc + P_smooth + P_bound
其中:
  R_acc = 10.0 × Σ_{i=0}^{13} w_i × exp(-2.0 × (v̂_i - v_set_i)²)
  w_i = [1.0, 0.9, ..., 0.1]  (时间衰减)
  P_smooth = -5.0 × Σ max(0, |Δalf|/alf_range - 3.0/alf_range)  (归一化日变化超标惩罚)
           + -5.0 × Σ max(0, |Δout|/out_range - 100/out_range)
  P_bound = -10 × 超出比例  (边界软约束)

关键修复：violation除以动作范围(ALF/15, OUT/700)，将两个动作的惩罚统一到[0,1]比例空间，
避免OUT的惩罚(-2260/step)碾压R_acc(+67 max)。修复后单步奖励在约[-30, +70]合理范围。
```

#### 3.2.4 `ppo参数优化/model/ppo.py` — MPD-PPO算法实现

**网络结构**：

**`Actor`**（策略网络）:
```
状态(7×feat_dim+14+1) → 拆分 →
  过去7天特征 → LSTM+槽嵌入 → 隐藏状态 ─┐
  目标电压(14d) ────────────────────────┤→ concat → SharedNet(Tanh) →
  槽嵌入 ───────────────────────────────┘          ├→ mean_head(28维) → tanh → [-1,1]
                                                   └→ std_head(28维) → softplus → (0,∞)
```

关键设计：
- `load_state_dict()` 内置旧版兼容映射（自动将4头→2头参数映射，跳过形状不匹配的参数）
- 输出28个独立均值+标准差（每维度独立高斯分布）
- 权重初始化：LSTM用xavier/orthogonal，线性层用orthogonal，`std_head` bias初始-1.0（初始std≈0.31，避免过度随机探索）

**`Critic`**（价值网络）：
```
与Actor共享相同的LSTM编码器结构，输出标量V(s)∈[-50,50]
（从[-1000,1000]收紧，防止值函数漂移导致GAE崩塌）
```

**`MPDPPO`**（算法主类）：

| 方法 | 功能 |
|------|------|
| `select_action(state)` | 采样动作(28维) + 对数概率 |
| `store_transition(...)` | 存储经验到缓冲区 |
| `compute_gae(values, next_values, dones)` | 广义优势估计 + 标准化 |
| `update(ppo_epochs=10)` | PPO更新（独立裁剪+求和） |
| `save_model(path)` / `load_model(path)` | 模型持久化 |

**差异化裁剪实现**（核心算法改进）：

```python
# 偶数索引(0,2,4,...,26)：ALF — 裁剪率0.1
# 奇数索引(1,3,5,...,27)：OUT — 裁剪率0.2
alf_surr = min(alf_ratio × A, clip(alf_ratio, 0.9, 1.1) × A)
out_surr = min(out_ratio × A, clip(out_ratio, 0.8, 1.2) × A)
actor_loss = -(alf_surr + out_surr)  # 独立裁剪+求和
```

**PPO更新流程**：
1. 对rewards做标准化（零均值单位方差，替代硬裁剪[-50,50]，保留相对好坏信号）
2. 计算GAE优势 → 标准化（若GAE坍缩则回退到TD-error，TD-error也坍缩则跳过更新）
3. 构建targets = advantages + values
4. 10轮内层迭代 × mini-batch(64)：
   - 独立计算ALF和OUT的ratio + clipped ratio
   - 分别取min后求和得actor_loss
   - MSE计算critic_loss
   - 梯度裁剪(max_norm=1.0) + 优化器step

**经验缓冲区清理**：每次update后自动清空（`_clear_buffers()`）

#### 3.2.5 `ppo参数优化/model/train_ppo.py` — PPO训练脚本

**数据加载** (`load_data_for_ppo()`):
- 关键改进：`target_voltage` 改为 `实际设定`（设定电压），而非历史工作电压
- 复用防御性编程：当 `实际设定` 列缺失时回退到 `TARGET`（工作平均）
- 输出：样本列表（含 `past_features`, `target_voltage`, `pot_id`, `pot_num`）

**训练循环**：
```
for epoch in range(1000):
    # 收集经验（累积256步），同时累加reward分量用于诊断
    while total_steps < 256:
        随机采样一个样本 → run_episode() → 存储经验 + 累加R_acc/P_smooth/P_bound
    # PPO更新（10轮内层迭代 × mini_batch=64）
    actor_loss, critic_loss = ppo_agent.update()
    # 每10轮打印: Reward: X (R_acc=Y P_sm=Z P_bd=W) | Actor: L | Critic: L
    # 仅保存最佳+最终模型
```

**`run_episode()` 函数**：单个episode（最多14步）的执行循环，返回总reward及各分量（R_acc/P_smooth/P_bound）用于日志诊断。

**模型保存**：仅保存最佳模型（`best_ppo_model.pth`）和最终模型（`final_ppo_model.pth`），移除了定期保存以减少磁盘占用。

**训练日志**：每10轮打印 avg reward 及其分量组成 `Reward: X (R_acc=Y P_sm=Z P_bd=W)`，便于快速诊断奖励函数行为。

### 4.2 模型间依赖关系

```
LSTMModelWithPotEmbedding (基础时序编码器)
    │
    ├─→ ConditionalVoltagePredictor (条件预测 = 基础编码器 + 条件编码器)
    │       │
    │       └─→ VoltageControlEnv (仿真环境，封装预测器为MDP)
    │               │
    │               └─→ MPDPPO (含Actor/Critic，在仿真环境中训练)
    │
    └─── Actor/Critic 内部也有独立的LSTM编码器（结构对称，参数独立）
```

**关键设计决策**：Actor/Critic的LSTM编码器与条件预测器的LSTM是**独立训练**的，不共享权重。虽结构对称，但各有不同的优化目标。

### 五.2 第二轮修复：训练稳定性Bug修复

训练过程中发现 reward 持续为负(-2000~-3200)且 Actor Loss 锁死为零的问题，经诊断发现3个Critical级bug：

| 修复 | 问题 | 修复内容 | 涉及文件 |
|------|------|---------|---------|
| 修复7 | target_voltage 逐步缩减导致形状崩溃 | 滑动窗口时用末值填充保持14维：`np.append(target[1:], target[-1])` | `environment.py:297` |
| 修复8 | 平滑惩罚量级爆炸(OUT -2260/step)碾压精度奖励 | violation 除以动作范围归一化到[0,1]比例空间，SMOOTH_VIOLATION_WEIGHT 1.0→5.0 | `environment.py:140-161`, `config.py:98` |
| 修复9 | GAE中reward硬裁剪[-50,50]抹除全部学习信号 | 移除硬裁剪，改用reward标准化（零均值单位方差） | `ppo.py:370-377` |
| 修复10 | 值函数漂移(+1000)导致GAE反向传播链式坍缩 | Critic输出clamp从[-1000,1000]收紧到[-50,50]，GAE坍缩时回退到TD-error | `ppo.py:278`, `ppo.py:382-394` |
| 修复11 | Actor初始std=0.694导致过度随机探索 | std_head bias初始化为-1.0（初始std≈0.31） | `ppo.py:113` |
| 修复12 | 缺乏奖励分量诊断手段 | info增加R_acc/P_smooth/P_bound分量，训练日志打印分量组成 | `environment.py:183,377`, `train_ppo.py:163-265` |

**根因链条**：平滑惩罚(OUT)量级放大 → reward全部极负 → reward硬裁剪抹除差异 → GAE崩塌 → Actor Loss=0 → 训练死锁。五项修复形成闭环。

## 六、项目优势总结

1. **迁移学习设计**：条件预测器的`ConditionalVoltagePredictor`架构通过冻结预训练LSTM + 新增轻量条件编码器，实现了零破坏的模型扩展
2. **训练-部署一致性**：Actor输出完整14天轨迹，消除"保持策略"导致的分布偏移，预测器输入在训练和推理时保持一致
3. **多步前瞻优化**：14天加权奖励让Agent考虑长期电压质量，而非贪心优化单日
4. **差异化裁剪**：ALF(ε=0.1)和出铝量(ε=0.2)分别控制更新幅度，符合实际操作的物理约束
5. **完备的评估体系**：预测模型有6种可视化 + 分日指标；PPO有18个测试样本的完整轨迹可视化
6. **工程化细节**：旧版模型兼容加载、NaN安全保护、梯度裁剪、边界检查、路径自适应

## 七、潜在改进方向

1. **预测模型精度提升**：当前条件预测器仅使用Conv1D编码未来动作（kernel_size=3），可尝试Transformer或LSTM编码动作序列
2. **经验更新系数标定**：铝水平/出铝量的经验系数(-0.3)需用实际数据校正，当前为经验估计
3. **多目标优化**：当前仅优化电压逼近设定值，可扩展为包含电流效率、能耗等多目标
4. **在线学习**：部署后收集真实反馈，定期微调预测模型和PPO策略以跟踪电解槽老化
5. **Actor/Critic与预测器权重共享**：可尝试冻结预测器LSTM→初始化Actor LSTM，加速PPO收敛
