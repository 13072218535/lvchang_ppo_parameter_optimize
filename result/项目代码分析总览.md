# 铝电解槽工作电压参数优化项目 — 代码分析总览 (2026-06-10更新)

## 一、项目概览

本项目针对铝电解槽的**工作电压控制优化**问题，构建了一套完整的"**条件电压预测 + 强化学习优化**"流水线。核心目标：通过智能调整 `ALF加料量` 和 `实际出铝量` 两个可控参数，使电解槽的**工作平均电压尽可能逼近设定电压**，从而提升电流效率和降低能耗。

项目已完成多轮迭代：
- **基础架构**：条件电压预测模型（LSTM+条件编码器）+ MPD-PPO（28维轨迹输出）
- **训练稳定性修复**：标准化闭环、奖励函数归一化、GAE防崩塌
- **多算法对比框架**：8种RL算法统一接口（On-policy × 5 + Off-policy × 3）
- **环境马尔可夫性修复**：状态扩展、加权动作转移、Off-policy专项优化
- **TAA-PPO改进算法**：时间自适应裁剪 + 预热调度 + 自适应平滑正则
- **TAA-PPO-4D V2/V3算法**：4维降维 + MLP Critic(V2) → LSTM Critic(V3，收敛修复)
- **对比可视化优化**：差异放大布局、双Y轴、偏离基线图
- **仿真器不确定性量化(UQ)**：5成员Deep Ensemble + 乘法P_unc惩罚项（λ=50）
- **对抗训练流水线**：agent探索数据增强预测器训练（全算法MAE下降29%）
- **对抗增强 vs 无增强对比实验**（2026-06-09）：增强数据训练集成学习器 → 8算法RL训练 → 对比评估
  - Off-policy 受益最大：DDPG +31.2%, TD3 +9.2%
  - 最终策略评估 MAE 11.7~12.1 mV，全面优于人类操作员（17.4 mV）+30~33%

## 三、模块A：条件电压预测模型（`model/`）

### 3.1 `model/config.py` — 预测模型配置

定义预测模型的所有超参数和数据处理参数：

| 参数类别 | 关键参数 | 说明 |
|---------|---------|------|
| 序列配置 | `INPUT_LEN=7`, `OUTPUT_LEN=14` | 输入7天，预测14天 |
| 模型结构 | `HIDDEN_DIM=128`, `NUM_LAYERS=2`, `POT_EMBED_DIM=16` | 2层LSTM + 槽号16维嵌入 |
| 训练控制 | `BATCH_SIZE=64`, `NUM_EPOCHS=100`, `LEARNING_RATE=0.001` | Adam优化器，ReduceLROnPlateau调度 |
| 早停 | `EARLY_STOPPING_PATIENCE=15` | 验证集loss不降15轮停止 |
| 数据划分 | `TRAIN_POTS`(30个槽), `VAL_POTS`(6个), `TEST_POTS`(6个) | 按槽号独立划分，避免数据泄露 |
| 特征选择 | `HIGH_CORR_FEATURES`(12个) | 基于Spearman相关系数筛选（\|r\|≥0.2） |

### 3.2 `model/data_processor.py` — 数据处理流水线

**类 `VoltageDataset`**：PyTorch Dataset封装，支持四种样本格式。

**类 `DataProcessor`**：6步数据处理流水线：
1. `load_data()` — 读取Excel，按日期排序
2. `preprocess_data()` — 缺失值处理（分组前向填充 + 线性插值 + 均值填充）
3. `create_features()` — 仅使用12个原始高相关性特征，不做统计/衍生特征（避免仿真中统计特征反馈回路）
4. 数据划分 — 支持按时间(`time`)或按槽号(`pot`)两种模式
5. 标准化 — 仅在训练集上fit scaler，然后transform全部
6. 序列构建 — 滑动窗口切分(7天输入 + 14天输出)

**特别设计**：`use_future_actions=True` 时额外提取未来14天的 `[ALF加料量(实际), 实际出铝量]` 动作序列，形状为 `(样本数, 14, 2)`，用于训练条件预测模型。

### 3.3 `model/model.py` — 神经网络定义

| 类名 | 层级 | 输入 | 输出 | 说明 |
|------|------|------|------|------|
| `LSTMModel` | 基础 | `(B,7,feat_dim)` | `(B,14)` | 纯LSTM，无槽嵌入 |
| `LSTMModelWithPotEmbedding` | 增强 | `(B,7,feat_dim)` + `(B,)` pot_ids | `(B,14)` | LSTM + 槽号16维嵌入 + BatchNorm |
| `ConditionalVoltagePredictor` | 条件 | `(B,7,feat_dim)` + `(B,14,2)` 动作 + pot_ids | `(B,14)` | 基础LSTM隐藏状态 + 条件动作LSTM编码 → concat → FC预测 |

**架构**：
```
过去7天特征 → 基础LSTM+槽嵌入 → hidden(128维) ──┐
                                                  ├→ concat(160维) → FC(64→32→14) → 未来14天电压
未来14天动作 → LSTM(2→16)→FC(32维) ──────────────┘
```

条件编码器使用单层LSTM替代原Conv1D+AvgPool，保留动作序列的完整时序信息。

### 3.4 `model/train.py` — 预测模型训练

- **评估函数**：`calculate_metrics()` (MAE/MSE/RMSE/MAPE/R²)、`evaluate_by_day()` (逐日评估)
- **训练引擎**：`train_epoch()` / `validate_epoch()` — 适配三种模型类型
- **预训练加载**：基础LSTM权重从 `best_model.pth` 加载，采用差分学习率
- **模型保存**：最佳模型(`best_model.pth`) + 条件最佳(`best_conditional_model.pth`) + 最终模型

### 3.5 `model/predict.py` — 预测推理

命令行推理工具：文件模式（读取CSV → 取最近7天 → 预测14天电压）和演示模式。

### 3.6 `model/ensemble_predictor.py` — 不确定性量化模块（新增 2026-06-02）

**类 `UncertaintyQuantifiedPredictor`**：Deep Ensemble 包装器
- 持有 N 个 `ConditionalVoltagePredictor` 实例（N 默认=5，不同随机种子训练）
- `predict(past_features, future_actions, pot_id)` → `(mean, variance, all_preds)`
  - mean: (B,14) 所有成员的均值预测
  - variance: (B,14) 样本方差（ddof=1），钳制 floor=1e-8
- `load_checkpoints(paths)` — 从多个 .pth 加载各成员权重，含 partial load 兼容
- `estimate_threshold(train_loader, percentile=95)` — 在训练集上校准不确定性基线

**类 `MCDropoutPredictor`**：轻量替代方案
- 单模型 + M 次 stochastic forward pass（dropout 开启）
- 无需重新训练，但可靠性低于 ensemble

### 3.7 `model/train_ensemble.py` — Ensemble 训练脚本（新增 2026-06-02）

- `--phase base`：训练 N 个基础 LSTM（使用不同 seed）
- `--phase conditional`：训练 N 个条件预测器（加载对应 base model）
- `--phase calibrate`：加载全部 checkpoint，在训练集上校准不确定性阈值
- `--use_augmented`：支持加载对抗增强数据（`adversarial_augmented.pkl`）
- 输出：`best_conditional_seed{N}.pth` + `final_conditional_seed{N}.pth` + `uncertainty_threshold.json`

**增强训练结果**（2026-06-09，`--use_augmented`）：
- 5成员 Mean Test MAE: **0.0870 ± 0.0033**，Mean R²: **0.9894**
- 训练数据: 6,289 样本（5,670 原始 + 619 对抗增强）
- P95 Uncertainty: 0.0478，OOD/ID Ratio: 1.44x
- 各 Seed MAE: 0.0815~0.0899，epochs 21~51（早停）

### 3.8 `model/adversarial_augment.py` — 对抗增强数据生成（新增 2026-06-02）

**三阶段流水线**：
1. **Agent探索**：加载 8 个已训练的 RL agent（mpd_ppo/vanilla_ppo/a2c/taa_ppo/taa_ppo_4d/ddpg/td3/sac），在训练槽数据上 rollout 生成动作序列
2. **伪标签打标**：用 5成员 Ensemble 为每个动作序列预测电压（伪标签）+ 计算 uncertainty variance
3. **盲点筛选**：筛选高不确定性样本（variance > P95 阈值），保存为 `adversarial_augmented.pkl`（619 样本）

**完整实验流程**（2026-06-09 执行）：
```
Phase A: 无增强基线
  ensemble_backup 权重 (6月2日) → RL训练8算法 → 基线Reward

Phase B: 对抗增强对比
  adversarial_augmented.pkl → train_ensemble --use_augmented → 
  增强集成权重 → RL训练8算法 → 增强Reward → 对比分析

Phase C: 策略评估 (visualize_compare.py)
  全部RL策略 → 测试槽评估 → MAE vs Human → Win Rate
```

### 3.9 `model/data_processor.py` — 新增增强数据加载

**新增方法 `load_augmented_sequences(aug_path, feature_cols)`**：
- 读取 `adversarial_augmented.pkl`，标准化 X/actions/y
- 返回可直接 concat 的 (X, y, future_actions, pot_ids) 元组

**修改 `process()`**：新增 `augmented_data_path` 参数，自动合并增强数据到训练集。

### 4.3 `environment.py` — 电压控制仿真环境（MDP核心）

**类 `VoltageControlEnv`** — 封装条件预测模型，实现标准强化学习环境接口。

#### 状态空间（102维，已修复马尔可夫性）

```
state = [past_features(84) | target_voltage(14) | last_action(2) | cumulative_error(1) | pot_id(1)]
```

- 过去7天完整特征窗口（7×12=84维，展平）
- 未来14天目标设定电压序列（14维）
- **上次执行的day-1动作**（2维，修复违反1：跨episode平滑约束不再依赖隐藏变量）
- **累积电压误差**（1维，修复违反2：done信号可预测）
- 槽号索引（1维）

#### 动作空间（28维）

14天×2动作的完整轨迹：`[alf₁, out₁, alf₂, out₂, ..., alf₁₄, out₁₄]`，归一化到[-1,1]

#### 核心方法

| 方法 | 功能 |
|------|------|
| `reset(past_features, target_voltage, pot_id)` | 初始化状态，返回s₀ |
| `step(action_trajectory_28d)` | 滚动时域执行：预测14天电压→计算奖励→仅执行day-1→加权更新状态 |
| `_calculate_reward(voltage_pred, target, action_traj)` | 三部分奖励：R_acc(指数精度) + P_smooth(平滑惩罚) + P_bound(边界接近惩罚) |
| `_update_state(future_actions_full, voltage_pred, prev)` | **14天加权动作转移**（day-1权重最高→day-14最低），含经验更新规则 |
| `_predict_voltage(past_features, future_actions, pot_id)` | 标准化→预测→反标准化闭环 |
| `set_smoothness_weight(weight)` | 动态调整平滑惩罚权重（off-policy课程学习用） |
| `get_state_dim()` / `get_action_dim()` | 维度查询 |

#### 奖励函数

```
R = R_acc + P_smooth + P_bound

R_acc = 8.0 × Σ w_i × exp(-15.0 × (v̂_i - v_set_i)²)     (14天时间加权)
P_smooth = -20.0 × Σ max(0, |Δ| - max_change) / range   (日变化超标惩罚)
         × 3.0（若|Δ|>2×max_change）                     (极端违规乘数)
P_bound = -10 × ratio                                    (边界10%区域内线性惩罚)
```

#### 状态转移流程

```
输入28维轨迹 → 重塑(14,2) → 反归一化+裁剪 → 标准化alf/out
  → 预测14天电压 → 反标准化(原始V) → 计算14天加权奖励
  → 14天加权动作更新状态(w=REWARD_TIME_WEIGHTS)
  → 仅day-1执行经验更新(铝水平等)
  → target用末值填充保持14维 → 返回(s', r, done, info)
```

**关键改进**：`_update_state`不再仅用day-1动作更新，而是用14天加权平均（权重与reward时间衰减一致），使s'编码完整计划信息。

### 4.4 `ppo.py` — MPD-PPO算法实现（改进版）

#### Actor网络

```
State(102) → 拆分 →
  过去7天特征(84) → LSTM(128,2层) → hidden(128) ─┐
  目标电压(14) ─────────────────────────────────┤→ concat(158)
  槽嵌入(16) ──────────────────────────────────┘      │
                                                      ↓
                                              SharedNet:
                                                FC(158→128)→Tanh
                                                FC(128→64)→Tanh
                                                      │
                              ┌───────────────────────┴───────────────────────┐
                         mean_head(64→28)                             std_head(64→28)
                            tanh→[-1,1]                              softplus→(0,∞)
```

- 输出28个独立均值+标准差（14天×2动作，每维独立高斯分布）
- `std_head` bias初始-1.0（初始std≈0.31）
- `load_state_dict()` 内置旧版兼容映射

#### Critic网络

与Actor共享LSTM编码器结构，输出标量V(s)，钳制[-50,50]防止值函数漂移。

#### MPDPPO算法主类

| 方法 | 功能 |
|------|------|
| `select_action(state)` | 采样动作(28维) + 对数概率 |
| `store_transition(...)` | 存储经验到缓冲区 |
| `compute_gae(values, next_values, dones)` | 广义优势估计 + reward标准化 + GAE坍缩回退 |
| `update(ppo_epochs=10)` | PPO更新（差异化裁剪 + 轨迹平滑正则） |
| `set_entropy_coef(coef)` | 动态调整熵系数（退火调度用） |
| `save_model(path)` / `load_model(path)` | 模型持久化 |

#### 改进：轨迹平滑正则化

```python
# 在Actor loss中直接约束14天mean输出的日间变化
means_reshaped = means.view(-1, 14, 2)           # (batch, 14, 2)
day_diffs = means_reshaped[:, 1:, :] - means_reshaped[:, :-1, :]
alf_smooth_loss = relu(|alf_diff| - ALF_THRESHOLD)².mean()
out_smooth_loss = relu(|out_diff| - OUT_THRESHOLD)².mean()
actor_loss += SMOOTH_REG_WEIGHT × (alf_smooth_loss + out_smooth_loss)
```

比reward中的平滑惩罚更高效：直接在策略层面约束，信号更直接。

#### 差异化裁剪

```python
# ALF(偶数索引): ε=0.1, OUT(奇数索引): ε=0.2
alf_surr = min(alf_ratio×A, clip(alf_ratio, 0.9, 1.1)×A).mean()
out_surr = min(out_ratio×A, clip(out_ratio, 0.8, 1.2)×A).mean()
actor_loss = -(alf_surr + out_surr)
```

## 五、模块C：多算法对比框架（`algorithms/`）

### 5.1 `algorithms/__init__.py` — 模块入口

- **`create_algorithm(algo_name, ...)`** — 工厂函数，根据名称创建算法实例（7种算法）
- **`ALGO_REGISTRY`** — 7种算法的名称注册表（含taa_ppo）

### 5.2 `algorithms/base_algorithm.py` — 基础设施

#### `BaseAlgorithm`（抽象基类）
```python
class BaseAlgorithm(ABC):
    select_action(state) → action        # 动作选择
    store_transition(s,a,r,lp,ns,done)   # 经验存储
    update() → (actor_loss, critic_loss) # 网络更新
    save_model(path) / load_model(path)  # 模型持久化
```

#### `ReplayBuffer` — 经验回放池
- 容量100,000 transitions
- `push()` / `sample(batch_size)` — 存储/采样

#### `QNetworkMLP` — 轻量级Q-Critic网络
```
state(102) + action(28) = 130 → FC(256)→ReLU→FC(256)→ReLU→FC(1)
```
10-50x faster than LSTM版本，适合off-policy高频更新。

#### 网络工厂函数
- `create_actor(input_dim, num_pots, action_dim, device)` — 创建LSTM Actor
- `create_critic(input_dim, num_pots, device)` — 创建LSTM V-Critic

### 5.3 `algorithms/trajectory_utils.py` — 轨迹工具

#### `generate_correlated_noise(size, rho, noise_std)`
生成AR(1)时序相关噪声：`n[t] = ρ·n[t-1] + √(1-ρ²)·ε[t]`（ρ=0.85保持日间平滑）

#### `generate_trajectory_noise(rho, noise_std)`
为28维轨迹生成时序相关噪声（ALF和OUT各14天独立但各自时序相关）

#### `TrajectoryExpander`
将4维动作 `[alf_start, alf_end, out_start, out_end]` 线性插值为28维轨迹。预计算14天均匀插值权重。

### 5.4 `algorithms/ppo_vanilla.py` — Vanilla PPO

与MPD-PPO的**唯一差异**：所有28维使用统一裁剪率ε=0.2（无差异化裁剪），标准PPO loss（统一clip→mean）。

### 5.5 `algorithms/a2c.py` — A2C

纯策略梯度：`loss = -log_prob × A`，无裁剪，含熵正则（β=0.01）。

### 5.6 `algorithms/ddpg.py` — DDPG（改进版）

- **ReducedActor4D**：输出4维 `[alf_start, alf_end, out_start, out_end]` → TrajectoryExpander → 28维
- **MLP Q-Critic**：QNetworkMLP(102+28→1)
- **探索**：AR(1)时序相关噪声（ρ=0.85, σ=0.15）
- **更新**：确定性策略梯度 + 软更新Target网络（τ=0.005）

### 5.7 `algorithms/td3.py` — TD3（改进版）

在DDPG基础上增加三项改进：
1. **Clipped Double Q-learning**：双Q网络取min抑制过估计
2. **Delayed Policy Updates**：Critic更新2次才更新1次Actor
3. **Target Policy Smoothing**：target动作加clip噪声平滑Q值估计

同样使用4维ReducedActor4D + 时序相关噪声。

### 5.8 `algorithms/sac.py` — SAC（改进版）

- **StochasticReducedActor4D**：输出(means_4d, stds_4d) → 采样→TrajectoryExpander→28维
- **双Q MLP**：QNetworkMLP×2
- **自动α调节**：target_entropy=-4（4维动作），自动温度系数
- **重参数化采样**：rsample() + log_prob

### 5.9 `algorithms/taa_ppo.py` — TAA-PPO（时间自适应退火PPO，**新增**）

MPD-PPO的改进衍生算法，从5个维度探索改进（3项确认有效）。

**核心理念**：将MPD-PPO中无意义的"按动作类型(ALF/OUT)差异化裁剪"替换为有意义的"按时间维度差异化裁剪"——近端天(实际执行)紧裁剪，远端天(计划意图)松裁剪。

**网络架构**：复用 `ppo.py` 中的 `Actor` 和 `Critic`（BatchNorm + 单Critic），保持已验证的稳定基础架构。

**3项核心改进**：

| 改进 | 实现 | 效果 |
|------|------|------|
| 时间自适应裁剪 | 14天×2=28维独立ε：day0-2(0.10)→day3-6(0.15)→day7-10(0.20)→day11-13(0.25) | 替换无意义ALF/OUT二分 |
| 裁剪预热调度 | warmup_factor=1.8→1.0，线性退火80轮 | 前期探索速度匹配Vanilla PPO |
| 自适应平滑正则 | 权重0.005(epoch<30)→线性增长→0.05(epoch≥100) | 消除前期reward-正则化冲突 |

**类 `TAAPPO`**：
- `set_epoch(epoch)` — 外部设置当前epoch，驱动预热/自适应调度
- `_get_effective_eps()` — 计算预热后的28维有效ε
- `_get_smooth_weight()` — 计算当前epoch的自适应平滑权重
- `update()` — 28维per-index PPO clip（`ratio[:,28]` vs `eps[28]` broadcast）+ 自适应平滑正则

**与MPD-PPO的关键差异**：
- MPD-PPO：按ALF/OUT拆分ratio → 两个独立min(clip) → 求和（loss计算分叉）
- TAA-PPO：28维统一ratio → per-index clip → mean（统一简洁，训练速度翻倍）

**实验结果**：Best 747.81, T_90%=11, σ=16.42, 466s。收敛速度持平Vanilla PPO，稳定性接近MPD-PPO，训练时间减半。

### 5.10 `algorithms/taa_ppo_4d.py` — TAA-PPO-4D V3（**2026-06-02 更新**）

将 TAA-PPO 的时间自适应裁剪与 4 维降维架构结合。**V3 将 MLP Critic 替换为 LSTM Critic，修复训练不收敛问题。**

**动机演变**：
- V2：MLP Critic（弱模型→被迫泛化）+ Critic Replay Buffer
- V3：发现 MLP Critic 无法可靠估计 V(s)，导致 Critic Loss 4000+、P_smooth=-500、奖励不收敛。替换为 ppo.Critic(LSTM+槽嵌入) 后全部改善

**架构（V3）**：

```
Actor: StochasticReducedActor4D(LSTM) → 4维(means, stds) → TrajectoryExpander → 28维
Critic: ppo.Critic(LSTM, 243K参数)  与MPD-PPO/TAA-PPO同等可靠
```

**4项核心设计**：

| 设计 | 说明 |
|------|------|
| 4维动作空间 | `[alf_start, alf_end, out_start, out_end]` → 线性插值 → 28维 |
| 4维 ε 裁剪 | `[0.10, 0.25, 0.10, 0.25]` — start 紧、end 松 |
| 裁剪预热 | warmup_factor 1.8→1.0（80 epochs） |
| LSTM Critic | 243K参数，与 MPD-PPO/TAA-PPO 同架构，可靠收敛 |

**V2→V3 改进效果**：

| 指标 | V2 (MLP Critic) | V3 (LSTM Critic) |
|------|----------------|-----------------|
| Critic Loss | 4000+ → 10 (震荡) | 0.69 → 1.0 (稳定) |
| P_smooth | -250 ~ -700 | 0 |
| P_bound | -80 ~ -710 | 0 |
| 最佳奖励 | 329 (不收敛) | 694 (收敛) |

### 8.11 第十一轮：根因诊断 + 可复现性修复（2026-06-10）

**问题**：6月9日对抗增强实验未复现6月2日的29% MAE改善效果，On-policy训练Reward反而退化。

**根因1 (致命): pot_id 格式不匹配**
- `adversarial_augment.py` 存储 0-indexed pot_ids (0~41)，而 `DataProcessor` 在数据划分阶段使用实际槽号 (1101~1142)
- 合并后 unique pot count = 42 + 25 = **67**（正常应为42）
- `pot_embedding` 从 42→67 维度不匹配 → base model 加载失败 → **随机初始化**
- 条件预测器在未训练的 base LSTM 上构建，效果大幅退化

**根因2 (致命): 实验设计与6月2日完全不同**

| 维度 | 6月2日实验 | 6月9日实验 |
|------|----------|----------|
| 预测器 | 训练增强预测器 | 训练增强预测器 |
| RL策略 | **保持不变** | **全部重新训练** |
| 评估 | 旧RL策略 + 新预测器 → MAE↓29% | 新RL策略 + 新预测器 → Reward退化 |

6月2日验证"预测器变好→评估更准"，6月9日变成"从头训练RL"——两个完全不同的问题。

**根因3: 跳过 `--phase base --use_augmented`**
- 增强数据改变了训练集分布（6,289 vs 5,670 samples）
- 但 base LSTM 未在增强数据上重新训练
- 条件预测器的基础编码器与增强数据分布不匹配

**修复措施**：

| # | 修复 | 文件 | 效果 |
|---|------|------|------|
| 1 | `adversarial_augment.py` 存储实际槽号 | 改为 `r['pot_num']` 替代 `r['pot_id']` | pot_ids 从 0-indexed → 实际槽号 (1101-1142) |
| 2 | `data_processor.py` 兼容新旧格式 | 检测 `max >= 1000` 区分实际槽号 vs 0-indexed | 向后兼容旧版 .pkl |
| 3 | 修补现有 `adversarial_augmented.pkl` | 0-indexed→实际槽号映射 | 现有数据可用 |
| 4 | 增加 `--phase base --use_augmented` | base LSTM 也在增强数据上训练 | base model 与增强数据分布匹配 |

**修复验证**（2026-06-10 重新训练）：

| 阶段 | 结果 | 关键验证 |
|------|------|---------|
| Phase base | Mean MAE 0.1008, pot count=**42** | pot 数正确！ |
| Phase conditional | Mean MAE 0.1047 | **base model 成功加载**（无 mismatch 错误） |
| Phase calibrate | P95=0.0379, OOD/ID=1.32x | 阈值正常 |

**可复现性保障**：

```bash
# 1. 正确的增强数据生成（修复后）
python adversarial_augment.py --max_samples_per_pot 5 --device cuda
# → 存储实际槽号，与 DataProcessor 一致

# 2. 正确的集成训练流程（三阶段，全部用增强数据）
python train_ensemble.py --phase base --num_seeds 5 --use_augmented --device cuda
python train_ensemble.py --phase conditional --num_seeds 5 --use_augmented --device cuda
```
