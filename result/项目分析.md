# 项目初步分析

## 项目概述

本项目面向**铝电解工业场景**，核心目标是通过**深度学习**与**强化学习**相结合的方式，实现电解槽工作电压的精准预测与工艺参数的智能优化。项目分为两大模块：

1. **条件电压预测模块** (`model/`) — 基于LSTM的时序预测模型，输入历史数据与未来操作动作，预测未来14天工作电压
2. **MPD-PPO参数优化模块** (`ppo参数优化/model/`) — 基于PPO强化学习算法，在仿真环境中优化ALF加料量和出铝量的操作策略

## 模块一：条件电压预测模块 (`model/`)

### 整体流程

```
原始数据 → 特征工程 → 序列化 → 模型训练 → 评估 → 推理预测
```

### 各文件详解

#### `config.py` — 全局配置
- **作用**：集中管理所有超参数、数据路径、特征列表
- **核心配置**：
  - `INPUT_LEN = 7`：输入过去7天数据
  - `OUTPUT_LEN = 14`：预测未来14天工作电压
  - `HIDDEN_DIM = 128`：LSTM隐藏层维度
  - `BATCH_SIZE = 64`、`NUM_EPOCHS = 100`
  - **高相关性特征列表** `HIGH_CORR_FEATURES`：基于Spearman相关系数筛选出的12个关键特征（如平均电压、电压设定、铝水平、Fe含量、槽龄等）
  - **槽号划分**：42个电解槽按`TRAIN_POTS(30)` / `VAL_POTS(6)` / `TEST_POTS(6)` 分组
- **依赖**：被所有其他模块导入

#### `data_processor.py` — 数据处理
- **类 `DataProcessor`**：完整的数据处理流水线
  - `load_data()`：从Excel读取原始数据，按槽号和日期排序
  - `preprocess_data()`：缺失值处理（前向填充+线性插值+均值填充），只保留高相关性特征
  - `create_features()`：**核心特征工程**，对每个基础特征生成6个衍生特征：
    - 3天/7天滑动窗口的均值与标准差
    - 1阶差分与7阶差分
    - 衍生特征：电压偏差、铝电解比例、槽龄_log/平方
    - 最终特征维度约84维
  - `create_sequences()`：创建滑动窗口序列，支持提取未来14天的动作序列（ALF加料量、实际出铝量）
  - `split_data()`：支持两种划分方式：
    - `time`：按时间顺序70%/17%/13%划分
    - `pot`：按预定义槽号分组划分
  - `process()`：端到端处理，返回DataLoader
- **类 `VoltageDataset`**：PyTorch数据集，支持返回 (X, y)、带槽号、带未来动作三种模式
- **依赖**：`config.py`

#### `model.py` — 模型定义
- **`LSTMModel`**：基础LSTM模型（无槽嵌入）
  - 结构：LSTM → BatchNorm → FC(128→64→32→14)
- **`LSTMModelWithPotEmbedding`**：**带槽嵌入的LSTM模型**
  - 对42个电解槽使用Embedding(42→16)向量表示
  - 槽嵌入与特征拼接后输入LSTM
  - 额外提供 `get_hidden()` 方法供条件预测器使用
- **`ConditionalVoltagePredictor`**：**条件电压预测器（核心创新）**
  - 接收：过去7天特征 + 未来14天动作序列(ALF, 出铝量) + 槽号
  - 流程：基础LSTM提取历史隐藏状态 → Conv1D编码未来动作 → 拼接 → FC预测头 → 14天电压
  - 支持迁移学习：可冻结基础LSTM参数，只训练新增部分
- **依赖**：`config.py`

#### `train.py` — 训练与评估
- **训练函数**：`train_epoch()` / `validate_epoch()` — 支持三种模式：
  - 基础LSTM（无槽嵌入）
  - 带槽嵌入LSTM
  - 条件预测模型（带未来动作）
- **评估函数**：
  - `calculate_metrics()`：MAE、MSE、RMSE、MAPE、R²
  - `evaluate_by_day()`：分日评估（14天的逐日指标）
  - `test_evaluate()`：测试集前向传播
- **可视化函数**：
  - `plot_losses()`：训练/验证损失曲线（含对数坐标）
  - `plot_metrics_by_day()`：分日MAE/RMSE/MAPE/R²柱状图
  - `plot_predictions()`：抽样样本时序对比（含误差填充）
  - `plot_all_test_predictions_scatter()`：全量散点对比（含拟合线）
  - `plot_all_test_predictions_timeseries()`：全量时序对比
  - `plot_test_predictions_combined()`：组合版（时序+散点）
- **运行逻辑**：
  1. 配置参数 → 2. 数据处理 → 3. 创建模型 → 4. 训练循环(含早停、学习率衰减) → 5. 测试评估 → 6. 可视化输出 → 7. 保存结果到 `output/`
- **依赖**：`config.py`、`model.py`、`data_processor.py`

#### `predict.py` — 推理预测
- 提供命令行接口：`python predict.py --input data.csv --output result.csv`
- 加载训练好的模型和scaler → 标准化输入 → 前向预测 → 保存CSV结果
- 支持演示模式（生成随机模拟数据）
- **依赖**：`config.py`、`model.py`

#### `output/` — 输出目录
- `best_model.pth`：最佳基础LSTM模型权重
- `best_conditional_model.pth`：最佳条件预测模型权重
- `final_conditional_model.pth`：最终条件预测模型权重
- `scaler.pkl`：StandardScaler对象
- `loss_curves.png`、`predictions_comparison.png` 等可视化图表
- `test_results.txt`：测试集评估结果文本

## 模块二：MPD-PPO参数优化模块 (`ppo参数优化/model/`)

### 整体流程

```
条件预测模型(已训练) → 构建仿真环境 → PPO Agent(环境交互) → 策略优化 → 保存策略
```

### 各文件详解

#### `config.py` — PPO专属配置
- 继承电压预测模块的全部配置，额外增加PPO参数：
  - `PPO_EPOCHS = 1000`：迭代次数
  - `PPO_GAMMA = 0.99`、`PPO_LAMBDA = 0.95`：折扣与GAE参数
  - `EPS_CLIP_ALF = 0.1`、`EPS_CLIP_OUT = 0.2`：**差异化裁剪阈值**（ALF变化小，出铝量变化大）
  - 奖励函数权重：`REWARD_ACC_WEIGHT = 10.0`、`REWARD_PROG_WEIGHT = 0.3` 等
  - 动作范围：`ACTION_ALF_MIN/MAX = [25, 40]`、`ACTION_OUT_MIN/MAX = [3500, 4200]`
  - 动作变化约束：`ACTION_ALF_MAX_CHANGE = 3.0`、`ACTION_OUT_MAX_CHANGE = 100.0`

#### `model.py` — 模型定义（与预测模块相同）
- 代码内容与 `model/model.py` 完全一致
- 提供 `LSTMModel`、`LSTMModelWithPotEmbedding`、`ConditionalVoltagePredictor` 三个类
- 在 `environment.py` 中用于加载训练好的条件预测器
- **依赖**：`config.py`

#### `environment.py` — 仿真环境
- **类 `VoltageControlEnv`**：MDP仿真环境
  - **状态设计**：完整7天历史窗口展平（7×feature_dim） + 目标14天电压序列 + 槽号索引
  - **动作空间**：2维连续动作[ALF加料量, 实际出铝量]，归一化到[-1,1]
  - **环境转移**：
    1. 接收动作 → 反归一化为实际值（含裁剪）
    2. 构建未来14天动作序列（保持策略：重复当天动作）
    3. 调用条件预测模型预测14天电压
    4. 取第1天预测电压计算奖励
    5. 滑动时间窗口，更新状态（更新统计特征、累计误差等）
  - **奖励函数**（四部分）：
    - `R_acc`：精度奖励，`10 * exp(-2 * e_t²)`，误差越小奖励越大
    - `R_prog`：进度奖励，`0.3 * (best_error - e_t)`，鼓励持续改进
    - `P_smooth`：平滑惩罚，动作变化超出约束时惩罚
    - `P_bound`：边界惩罚，动作超出合法范围时按比例惩罚
  - **终止条件**：超过14步 或 累计误差 > 0.3V
- **`load_predictor_model()`**：加载训练好的条件电压预测模型
- **依赖**：`model.py`、`config.py`

#### `ppo.py` — PPO算法实现
- **类 `Actor`**：**多分支策略网络**
  - LSTM编码器处理7天历史窗口 → 槽嵌入拼接 → 共享网络 → 两个独立分支输出ALF和出铝量的均值/标准差
  - 输出通过tanh限制在[-1,1]，std通过Softplus保证正数
  - 支持 `sample_action()`（环境交互用，无梯度）和 `get_log_prob()`（更新用，有梯度）
- **类 `Critic`**：**价值网络**
  - 与Actor共享相同的LSTM编码器结构
  - 输出标量状态价值 V(s)
- **类 `MPDPPO`**：**PPO算法封装**
  - `select_action()`：选择动作并存储经验
  - `compute_gae()`：广义优势估计
  - `update()`：核心更新逻辑，支持：
    - 差异化裁剪（ALF用0.1，出铝量用0.2）
    - PPO代理目标函数（clipped surrogate objective）
    - 梯度裁剪（max_norm=1.0）
    - 全面的NaN/Inf防御
  - `save_model()` / `load_model()`：模型持久化
- **依赖**：`config.py`

#### `train_ppo.py` — PPO训练脚本
- **`create_features_for_ppo()`**：特征工程（与 `data_processor.py` 保持一致）
- **`load_data_for_ppo()`**：加载数据，创建训练样本，复用预测模块的scaler
- **`run_episode()`**：运行单个episode（14步）
- **训练主循环**：
  1. 设置随机种子和设备
  2. 加载训练样本
  3. 加载条件预测模型并构建仿真环境
  4. 初始化PPO Agent
  5. 训练循环（最多1000轮）：
     - 收集经验直到达到 `PPO_BATCH_SIZE`
     - 更新PPO网络
     - 保存最佳模型和定期检查点
     - 每10轮打印日志
- **依赖**：`config.py`、`model.py`、`environment.py`、`ppo.py`

## 关键设计特点

1. **槽嵌入（Pot Embedding）**：将42个电解槽映射为16维稠密向量，让模型学习不同槽位的个体特性
2. **条件预测架构**：将"未来操作动作"作为条件输入，使模型能回答"如果这样操作，电压会变成多少"
3. **迁移学习**：基础LSTM预训练后冻结，仅训练条件编码器，减少训练成本
4. **差异化PPO裁剪**：对ALF加料量（`ε=0.1`）和出铝量（`ε=0.2`）使用不同裁剪阈值，精细控制策略更新步长
5. **仿真环境MDP设计**：滑动窗口状态更新 + 保持策略的未来动作填充，使强化学习在离线数据上也能训练

## 后续可能的优化方向

- 未来动作序列改为策略自洽填充（而非简单的保持策略）
- 共享预测模型与Actor网络的LSTM权重
- 批量经验收集时正确处理episode边界（GAE计算）
- 调参优化：奖励函数权重、学习率、网络结构等
